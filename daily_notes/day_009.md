## 1. The Optimization Problem

### 1.1 What We Are Solving

Deep learning training is an **optimization problem**: find parameters $\theta$ that minimize
the empirical loss over a dataset $\mathcal{D} = \{(\mathbf{x}_i, y_i)\}_{i=1}^N$:

$$\theta^* = \arg\min_\theta \mathcal{L}(\theta) = \arg\min_\theta \frac{1}{N}\sum_{i=1}^N \ell(f(\mathbf{x}_i;\theta),\, y_i)$$

where:
- $f(\mathbf{x};\theta)$ is the neural network with parameters $\theta$
- $\ell(\cdot,\cdot)$ is the per-sample loss (e.g., cross-entropy, MSE)
- $\mathcal{L}(\theta)$ is the full-batch loss (average over all samples)

---

### 1.2 Why This Is Hard

**Non-convex:** $\mathcal{L}(\theta)$ has many local minima, saddle points, and flat regions.
Unlike convex functions (unique global minimum), gradient descent is not guaranteed to find
the best solution.

**High-dimensional:** $\theta \in \mathbb{R}^d$ where $d$ can be billions (GPT-4: ~1.8T parameters).
Computing and storing the full Hessian $H = \nabla^2\mathcal{L} \in \mathbb{R}^{d\times d}$ is impossible.

**Noisy gradients:** we estimate $\nabla\mathcal{L}(\theta)$ from mini-batches, not the full dataset.
The mini-batch gradient is an unbiased but noisy estimate of the true gradient.

**Ill-conditioning:** the loss surface can have very different curvature in different directions.
Condition number $\kappa = \lambda_{\max}(H)/\lambda_{\min}(H) \gg 1$ makes gradient descent slow.

---

### 1.3 The Gradient

The gradient $\nabla_\theta\mathcal{L}(\theta) \in \mathbb{R}^d$ points in the direction of
**steepest ascent** of $\mathcal{L}$.

Moving in the **negative gradient direction** decreases $\mathcal{L}$:

$$\mathcal{L}(\theta - \eta\nabla\mathcal{L}) \approx \mathcal{L}(\theta) - \eta\|\nabla\mathcal{L}\|^2 < \mathcal{L}(\theta)$$

This approximation holds for small $\eta$ — the first-order Taylor expansion:

$$\mathcal{L}(\theta + \Delta\theta) \approx \mathcal{L}(\theta) + \nabla\mathcal{L}(\theta)^\top\Delta\theta + \frac{1}{2}\Delta\theta^\top H\Delta\theta$$

Setting $\Delta\theta = -\eta\nabla\mathcal{L}$ gives descent (for small enough $\eta$).

---

### 1.4 Critical Points

A point $\theta^*$ is a **critical point** if $\nabla\mathcal{L}(\theta^*) = 0$.

| Type | Hessian eigenvalues | Description |
|---|---|---|
| Local minimum | All $\lambda_i > 0$ | $H$ positive definite — loss increases in every direction |
| Local maximum | All $\lambda_i < 0$ | $H$ negative definite |
| Saddle point | Mixed signs | Minimum in some directions, maximum in others |
| Flat region | Many $\lambda_i \approx 0$ | Plateau — gradient near zero but not at a minimum |

**In high dimensions:** saddle points are exponentially more common than local minima (Dauphin et al. 2014).
A point with random sign eigenvalues in $d$ dimensions: probability of all positive $= 2^{-d}$.

SGD noise actually helps escape saddle points — gradient estimates don't all point to the
saddle, so the trajectory drifts away naturally.

## 2. Gradient Descent — Core Idea

### 2.1 Full-Batch Gradient Descent

Compute the gradient over the **entire dataset**, then update:

$$\mathbf{g}_t = \nabla_\theta\mathcal{L}(\theta_t) = \frac{1}{N}\sum_{i=1}^N \nabla_\theta\ell(f(\mathbf{x}_i;\theta_t), y_i)$$

$$\theta_{t+1} = \theta_t - \eta\mathbf{g}_t$$

**Properties:**

| Property | Value |
|---|---|
| Gradient estimate | Exact (full dataset) |
| Memory | Must store all $N$ gradients |
| Computation per step | $O(N)$ — expensive |
| Steps per epoch | 1 |
| Convergence | Monotone decrease for convex loss |

**The descent lemma:** for $L$-smooth $\mathcal{L}$ (Lipschitz gradient with constant $L$):

$$\mathcal{L}(\theta_{t+1}) \leq \mathcal{L}(\theta_t) - \eta\left(1 - \frac{\eta L}{2}\right)\|\nabla\mathcal{L}(\theta_t)\|^2$$

For $\eta < 2/L$: the right side $< \mathcal{L}(\theta_t)$ — guaranteed descent per step.
Optimal step size: $\eta^* = 1/L$.

---

### 2.2 The Quadratic Case — Exact Analysis

For a quadratic loss $\mathcal{L}(\theta) = \frac{1}{2}\theta^\top H\theta - \mathbf{b}^\top\theta$
(optimal at $\theta^* = H^{-1}\mathbf{b}$):

$$\nabla\mathcal{L}(\theta) = H\theta - \mathbf{b} = H(\theta - \theta^*)$$

GD update: $\theta_{t+1} - \theta^* = (I - \eta H)(\theta_t - \theta^*)$

After $T$ steps: $\theta_T - \theta^* = (I - \eta H)^T(\theta_0 - \theta^*)$

In the eigenbasis of $H$ (eigenvalues $\lambda_1 \geq \lambda_2 \geq \cdots \geq \lambda_d > 0$):

$$e_T^{(i)} = (1 - \eta\lambda_i)^T e_0^{(i)}$$

**Convergence requirement:** $|1 - \eta\lambda_i| < 1$ for all $i$, i.e., $0 < \eta < 2/\lambda_1$.

**Convergence rate:** dominated by the slowest dimension $\lambda_d$:

$$\|\theta_T - \theta^*\|^2 \leq \left(1 - \frac{2\eta\lambda_1\lambda_d}{\lambda_1+\lambda_d}\right)^T\|\theta_0 - \theta^*\|^2$$

With optimal $\eta^* = 2/(\lambda_1 + \lambda_d)$:

$$\text{rate} = \left(\frac{\kappa-1}{\kappa+1}\right)^2, \quad \kappa = \lambda_1/\lambda_d$$

**Ill-conditioning ($\kappa \gg 1$):** rate $\approx 1 - 4/\kappa$ — convergence is very slow.

---

## 3. Stochastic Gradient Descent (SGD)

### 3.1 The Key Idea

Instead of computing the full-dataset gradient (expensive), estimate it from a **mini-batch**
$\mathcal{B} \subset \mathcal{D}$ of size $B \ll N$:

$$\mathbf{g}_t = \frac{1}{B}\sum_{i\in\mathcal{B}_t}\nabla_\theta\ell(f(\mathbf{x}_i;\theta_t), y_i)$$

$$\theta_{t+1} = \theta_t - \eta\mathbf{g}_t$$

**Unbiased:** $\mathbb{E}_{\mathcal{B}}[\mathbf{g}_t] = \nabla\mathcal{L}(\theta_t)$

The mini-batch gradient is an unbiased estimator of the true gradient — it points in the right
direction on average, but with random noise around it.

---

### 3.2 Why Stochasticity Helps

**Speed:** $B \ll N$ gradient computations per step vs $N$ — many more steps per unit time.

**Escaping local minima:** gradient noise means SGD doesn't follow the gradient exactly.
The trajectory is a random walk biased toward lower loss — noise can push it over barriers.

**Implicit regularisation:** SGD noise acts as a regulariser. The "flat minima" that SGD tends
to find generalise better than the sharp minima that GD or small-batch training might find.

**Online learning:** can process streaming data without storing the whole dataset.

---

### 3.3 Bias-Variance of the Gradient Estimate

$$\text{Var}(\mathbf{g}_t) = \frac{\sigma^2}{B}$$

where $\sigma^2 = \text{Var}_{(\mathbf{x},y)\sim\mathcal{D}}[\nabla_\theta\ell(f(\mathbf{x};\theta),y)]$ is the per-sample gradient variance.

**Batch size effects:**

| $B$ | Gradient noise $\propto$ | Steps/epoch | Compute/step | Effect |
|---|---|---|---|---|
| 1 | $\sigma^2$ | $N$ | Minimal | Very noisy, slow convergence |
| 32–256 | $\sigma^2/B$ | $N/B$ | GPU-efficient | Standard — best trade-off |
| $N$ | 0 | 1 | Full dataset | Exact, but slow per epoch |

**The linear scaling rule:** doubling $B$ halves gradient noise → can double $\eta$:

$$\eta_{\text{new}} = \eta_{\text{base}} \cdot \frac{B_{\text{new}}}{B_{\text{base}}}$$

Valid for moderate batch sizes. Breaks down for very large $B$ (need warmup).

---

### 3.4 SGD Convergence for Convex Functions

For $\mu$-strongly-convex, $L$-smooth $\mathcal{L}$ with mini-batch variance $\sigma^2/B$:

$$\mathbb{E}[\mathcal{L}(\theta_T) - \mathcal{L}(\theta^*)] \leq \frac{L\|\theta_0-\theta^*\|^2}{2}\cdot\frac{1}{T} + \frac{\eta\sigma^2}{2\mu B}$$

Two terms:
- **Optimization error:** decays as $1/T$ — more steps help
- **Noise floor:** $\propto \eta\sigma^2/(\mu B)$ — irreducible with constant $\eta$

**Consequence:** to reach $\epsilon$ accuracy, must decay $\eta \to 0$. Optimal rate with
schedule $\eta_t = O(1/t)$: convergence $O(1/\sqrt{T})$ for non-convex.

---

### 3.5 SGD Implementation from Scratch

```python
import numpy as np


class SGD:
    """
    Vanilla Stochastic Gradient Descent.

    Update rule:
        theta <- theta - lr * grad

    Args:
        params:       list of numpy arrays (parameters to optimise)
        lr:           learning rate (step size)
        weight_decay: L2 regularisation coefficient (adds lr * wd * theta to grad)
    """
    def __init__(self, params: list, lr: float = 0.01,
                 weight_decay: float = 0.0):
        self.params       = params
        self.lr           = lr
        self.weight_decay = weight_decay

    def zero_grad(self):
        """Clear all stored gradients."""
        for p in self.params:
            p.grad = np.zeros_like(p.data)

    def step(self):
        """Apply one gradient descent update."""
        for p in self.params:
            if p.grad is None:
                continue

            grad = p.grad

            # L2 regularisation: add wd * theta to gradient
            if self.weight_decay != 0:
                grad = grad + self.weight_decay * p.data

            # Parameter update
            p.data -= self.lr * grad

    def __repr__(self):
        return (f"SGD(lr={self.lr}, weight_decay={self.weight_decay})")
```

## 4. SGD with Momentum

### 4.1 The Problem SGD Has

Vanilla SGD updates each parameter by the same (scaled) gradient at every step.

**Two failure modes:**

**1. Oscillation in ravines:** the loss surface is a steep narrow valley — gradient across the
valley is large (oscillates left-right), gradient along the valley is small (slow progress).

```
         ↑ large gradient component — oscillates
 loss    │╲    ╱╲    ╱╲
surface  │  ╲╱    ╲╱    ╲╱  ← zig-zag across ravine
         │──────────────────→ small gradient — slow progress
```

**2. Slow escape from flat regions:** gradient is tiny near plateaus — SGD barely moves.

**Solution:** accumulate a velocity vector that smooths out oscillations and builds up speed
in consistent gradient directions.

---

### 4.2 Heavy Ball / Classical Momentum

Inspired by physics: a ball rolling down a hill accumulates momentum — it keeps going even
when the slope is small, and the oscillations damp out.

$$\mathbf{v}_t = \beta\mathbf{v}_{t-1} + \eta\mathbf{g}_t$$

$$\theta_{t+1} = \theta_t - \mathbf{v}_t$$

Equivalently, written as update accumulation:

$$\mathbf{v}_t = \beta\mathbf{v}_{t-1} + (1-\beta)\mathbf{g}_t \quad\text{(then: }\theta_{t+1}=\theta_t - \frac{\eta}{1-\beta}\mathbf{v}_t\text{)}$$

The two conventions differ only in scaling. PyTorch uses the first form.

---

### 4.3 What Momentum Computes

Unrolling the recurrence:

$$\mathbf{v}_t = \eta\sum_{k=0}^{t}\beta^{t-k}\mathbf{g}_k$$

The velocity is a **geometrically-weighted sum of all past gradients** — an exponential moving
average (EMA) of gradients, biased toward recent ones.

**Effective learning rate analysis:**

If $\mathbf{g}_t = \mathbf{g}$ (constant gradient), at steady state:

$$\mathbf{v}_\infty = \frac{\eta\mathbf{g}}{1-\beta}$$

For $\beta=0.9$: velocity is $10\times$ the single-step gradient — 10× effective learning rate
in the consistent direction. In oscillating directions, the terms cancel:

$$\text{If gradient alternates } \pm\mathbf{g}: \quad \mathbf{v}_\infty \approx 0$$

This is the damping effect — momentum naturally kills oscillations.

---

### 4.4 Momentum in the Quadratic Case

For quadratic $\mathcal{L} = \frac{1}{2}\theta^\top H\theta$, momentum GD with optimal settings:

$$\text{Convergence rate} = \left(\frac{\sqrt{\kappa}-1}{\sqrt{\kappa}+1}\right)^2$$

Compare to GD rate $\left(\frac{\kappa-1}{\kappa+1}\right)^2$.

For large $\kappa$: GD rate $\approx 1 - 4/\kappa$, Momentum rate $\approx 1 - 4/\sqrt{\kappa}$.

**Speed-up:** $O(\sqrt{\kappa})$ fewer iterations to reach the same accuracy. Momentum is
**optimally fast** for quadratic objectives (matches Chebyshev polynomial lower bound).

**Optimal momentum parameter:**

$$\beta^* = \left(\frac{\sqrt{\kappa}-1}{\sqrt{\kappa}+1}\right)^2, \qquad \eta^* = \frac{4}{(\sqrt{\lambda_1}+\sqrt{\lambda_d})^2}$$

In practice: $\beta = 0.9$ is a reliable default (corresponds to $\kappa \approx 100$).

---

### 4.5 Implementation

```python
class SGDMomentum:
    """
    SGD with Heavy Ball Momentum.

    Update rule:
        v_t  = beta * v_{t-1} + lr * grad_t
        theta = theta - v_t

    At steady state with constant gradient g:
        v_inf = lr * g / (1 - beta)   (amplified by 1/(1-beta))

    Args:
        params:       list of parameters (numpy arrays with .data and .grad)
        lr:           learning rate
        momentum:     beta — EMA decay for velocity (0.9 is standard)
        weight_decay: L2 regularisation
        dampening:    reduces gradient contribution: v = beta*v + (1-dampening)*g
        nesterov:     use Nesterov variant (see Section 5)
    """
    def __init__(self, params: list, lr: float = 0.01,
                 momentum: float = 0.9, weight_decay: float = 0.0,
                 dampening: float = 0.0, nesterov: bool = False):
        self.params       = params
        self.lr           = lr
        self.beta         = momentum
        self.weight_decay = weight_decay
        self.dampening    = dampening
        self.nesterov     = nesterov

        # Initialise velocity buffers to zero
        self.velocity = [np.zeros_like(p.data) for p in params]

    def zero_grad(self):
        for p in self.params:
            p.grad = np.zeros_like(p.data)

    def step(self):
        for i, p in enumerate(self.params):
            if p.grad is None:
                continue

            grad = p.grad.copy()

            # L2 regularisation
            if self.weight_decay != 0:
                grad += self.weight_decay * p.data

            # Momentum update
            if self.beta != 0:
                self.velocity[i] = (self.beta * self.velocity[i]
                                    + (1 - self.dampening) * grad)
                if self.nesterov:
                    # Nesterov: look-ahead gradient
                    grad = grad + self.beta * self.velocity[i]
                else:
                    grad = self.velocity[i]

            p.data -= self.lr * grad

    def __repr__(self):
        return (f"SGDMomentum(lr={self.lr}, momentum={self.beta}, "
                f"nesterov={self.nesterov})")
```

### 4.6 Visualising Momentum

On a 2D quadratic with ill-conditioning ($\kappa = 10$):

```
                SGD (zig-zag)          SGD + Momentum (smooth)
                
  loss ↑         ↗↘                    loss ↑     ╭────────
  contour│     ↗    ↘↗                 contour│  ╭╯
  lines  │   ↗          ↘             lines  │ ╭╯
  (narrow│  ↗              ↘                 │╭╯
  valley)│  ↘              ↗               ──┤╯
         └──────────────────              ───┴──────────────

Many steps to reach minimum           Fewer steps, less oscillation
```

The momentum trajectory curves toward the minimum — it builds speed along the valley and
damps oscillations across it.

## 5. Nesterov Accelerated Gradient

### 5.1 The Lookahead Trick

In classical momentum, we compute the gradient at the current position $\theta_t$, then
move. Nesterov suggests: compute the gradient at the **lookahead position** where we expect
to be after applying the current velocity.

**Standard momentum:**
1. Compute $\mathbf{g}_t = \nabla\mathcal{L}(\theta_t)$
2. Update velocity: $\mathbf{v}_t = \beta\mathbf{v}_{t-1} + \eta\mathbf{g}_t$
3. Update params: $\theta_{t+1} = \theta_t - \mathbf{v}_t$

**Nesterov:**
1. Compute lookahead: $\tilde{\theta}_t = \theta_t - \beta\mathbf{v}_{t-1}$
2. Compute $\mathbf{g}_t = \nabla\mathcal{L}(\tilde{\theta}_t)$ ← gradient at lookahead
3. Update velocity: $\mathbf{v}_t = \beta\mathbf{v}_{t-1} + \eta\mathbf{g}_t$
4. Update params: $\theta_{t+1} = \theta_t - \mathbf{v}_t$

---

### 5.2 Equivalent Reformulation

Re-parameterise as $\phi_t = \theta_t - \beta\mathbf{v}_{t-1}$ (the lookahead position).
Then the Nesterov update in $\phi$ coordinates:

$$\mathbf{v}_t = \beta\mathbf{v}_{t-1} + \eta\nabla\mathcal{L}(\phi_t)$$

$$\phi_{t+1} = \phi_t - \beta^2\mathbf{v}_{t-1} - (1+\beta)\eta\nabla\mathcal{L}(\phi_t)$$

In PyTorch's implementation (used in `SGDMomentum` above with `nesterov=True`):

$$\mathbf{g}_t^{\text{NAG}} = \mathbf{g}_t + \beta\mathbf{v}_{t-1}$$

$$\theta_{t+1} = \theta_t - \eta\mathbf{g}_t^{\text{NAG}}$$

---

### 5.3 Why Nesterov is Better

Classical momentum: corrects after overshooting — the gradient at $\theta_t$ doesn't know
we're about to apply a large velocity step.

Nesterov: corrects before overshooting — sees the gradient at where we're going, not where
we are. The correction happens at the right time.

**Convergence rate (strongly convex):** Nesterov achieves $O(1/T^2)$ convergence vs $O(1/T)$
for gradient descent and $O(1/T^2)$ for optimal momentum — they match asymptotically, but
Nesterov has better constants in practice.

```python
# PyTorch usage:
optimizer = torch.optim.SGD(
    model.parameters(),
    lr=0.01,
    momentum=0.9,
    nesterov=True,    # enable Nesterov
)
```

## 6. Adagrad

### 6.1 Motivation: Per-Parameter Learning Rates

Vanilla SGD uses the same learning rate $\eta$ for every parameter. But parameters may need
very different step sizes:

- **Frequent features** (embeddings of common words): updated often → gradients accumulate →
  should use a smaller effective learning rate
- **Rare features** (embeddings of rare words): updated rarely → should use a larger effective
  learning rate

**Adagrad** (Duchi et al. 2011) adapts $\eta$ per parameter based on its history of gradients.

---

### 6.2 Update Rule

Accumulate squared gradients:

$$G_t = \sum_{k=1}^t \mathbf{g}_k^2 \quad\text{(element-wise sum of squares)}$$

Update with scaled learning rate:

$$\theta_{t+1} = \theta_t - \frac{\eta}{\sqrt{G_t + \epsilon}}\odot\mathbf{g}_t$$

The division is **element-wise** — each parameter gets its own effective learning rate.

---

### 6.3 Intuition

A parameter with large historical gradients (frequent, large updates) gets a small effective LR:

$$\eta_{\text{eff},j} = \frac{\eta}{\sqrt{G_{t,j}+\epsilon}} \approx \frac{\eta}{\sqrt{t}\cdot\text{RMS}(g_j)}$$

A parameter that rarely receives large gradients gets a large effective LR. This is automatic
curriculum — the optimizer focuses more on dimensions where progress is slow.

---

### 6.4 Problem: Monotonically Decreasing LR

$G_t$ only increases (sum of squares). So $\eta_{\text{eff},j} \to 0$ as $t \to \infty$ — 
the learning rate **never recovers** even if gradients become large again.

This makes Adagrad poorly suited for non-convex optimization (deep learning) — training
often stalls before convergence.

```python
class Adagrad:
    """
    Adaptive Gradient Algorithm.
    
    Accumulates squared gradients: G_t += g_t^2
    Update: theta -= (lr / sqrt(G_t + eps)) * g_t
    
    Problem: G_t only grows → effective LR → 0. Use RMSProp instead.
    """
    def __init__(self, params, lr=0.01, eps=1e-8, weight_decay=0.0):
        self.params       = params
        self.lr           = lr
        self.eps          = eps
        self.weight_decay = weight_decay
        self.G = [np.zeros_like(p.data) for p in params]   # accumulated sq grads

    def zero_grad(self):
        for p in self.params:
            p.grad = np.zeros_like(p.data)

    def step(self):
        for i, p in enumerate(self.params):
            if p.grad is None:
                continue
            grad = p.grad + self.weight_decay * p.data

            # Accumulate squared gradients
            self.G[i] += grad ** 2

            # Adaptive update
            p.data -= self.lr / (np.sqrt(self.G[i]) + self.eps) * grad
```

## 7. RMSProp

### 7.1 Fix Adagrad's Dying LR

**RMSProp** (Hinton, 2012 — unpublished lecture slides) replaces the cumulative sum of
squared gradients with an **exponential moving average**:

$$\mathbf{v}_t = \rho\mathbf{v}_{t-1} + (1-\rho)\mathbf{g}_t^2$$

$$\theta_{t+1} = \theta_t - \frac{\eta}{\sqrt{\mathbf{v}_t + \epsilon}}\odot\mathbf{g}_t$$

The EMA **forgets** old gradient information with decay $\rho$ (typically $0.9$–$0.99$).
This prevents the denominator from growing without bound.

---

### 7.2 What the EMA Computes

Unrolling the recurrence:

$$\mathbf{v}_t = (1-\rho)\sum_{k=0}^{t}\rho^{t-k}\mathbf{g}_k^2$$

This is a **geometric series** approximating the recent mean squared gradient:

$$\mathbf{v}_t \approx \mathbb{E}[\mathbf{g}^2]_{\text{recent}} = \text{RMS}(\mathbf{g})^2$$

So the effective learning rate is:

$$\eta_{\text{eff},j} \approx \frac{\eta}{\text{RMS}(g_j)_{\text{recent}}}$$

Parameters with large RMS gradient → small step. Parameters with small RMS gradient →
large step. Automatically normalises gradient scale.

---

### 7.3 Implementation

```python
class RMSProp:
    """
    RMSProp — Root Mean Square Propagation.
    
    Maintains EMA of squared gradients: v_t = rho*v_{t-1} + (1-rho)*g_t^2
    Update: theta -= (lr / sqrt(v_t + eps)) * g_t
    
    Fixes Adagrad's monotonically-decreasing LR via the EMA window.
    
    Args:
        params:       list of parameters
        lr:           global learning rate (alpha)
        rho:          EMA decay for squared gradients (0.9 typical)
        eps:          numerical stability (prevents div by zero)
        momentum:     optional momentum on the update (0 = disabled)
        weight_decay: L2 regularisation
        centered:     if True, subtract mean of g^2 (centred RMSProp)
    """
    def __init__(self, params, lr=0.01, rho=0.99, eps=1e-8,
                 momentum=0.0, weight_decay=0.0, centered=False):
        self.params       = params
        self.lr           = lr
        self.rho          = rho
        self.eps          = eps
        self.momentum     = momentum
        self.weight_decay = weight_decay
        self.centered     = centered

        self.v  = [np.zeros_like(p.data) for p in params]  # EMA of g^2
        self.buf= [np.zeros_like(p.data) for p in params]  # momentum buffer
        if centered:
            self.g_avg = [np.zeros_like(p.data) for p in params]  # EMA of g

    def zero_grad(self):
        for p in self.params:
            p.grad = np.zeros_like(p.data)

    def step(self):
        for i, p in enumerate(self.params):
            if p.grad is None:
                continue

            grad = p.grad + self.weight_decay * p.data

            # ── EMA of squared gradients ──────────────────────────────────────
            self.v[i] = self.rho * self.v[i] + (1 - self.rho) * grad ** 2

            if self.centered:
                # Centred RMSProp: subtract EMA of gradient (reduces variance)
                self.g_avg[i] = self.rho * self.g_avg[i] + (1 - self.rho) * grad
                denom = np.sqrt(self.v[i] - self.g_avg[i]**2 + self.eps)
            else:
                denom = np.sqrt(self.v[i] + self.eps)

            # ── Compute update ────────────────────────────────────────────────
            if self.momentum > 0:
                self.buf[i] = self.momentum * self.buf[i] + grad / denom
                p.data     -= self.lr * self.buf[i]
            else:
                p.data     -= self.lr * grad / denom
```

---

### 7.4 Centred RMSProp

Subtract the squared mean to get the **variance** of gradients:

$$\bar{\mathbf{g}}_t = \rho\bar{\mathbf{g}}_{t-1} + (1-\rho)\mathbf{g}_t$$

$$\text{Var}(\mathbf{g}_t) \approx \mathbf{v}_t - \bar{\mathbf{g}}_t^2$$

Using variance instead of second moment: normalises by the spread of gradients rather than
their raw magnitude. More stable when gradients have large mean.

## 8. Adam

### 8.1 Combining Momentum and RMSProp

**Adam** (Kingma & Ba, 2014 — one of the most cited papers in ML) combines:
- **First moment** (like momentum): EMA of gradients → direction
- **Second moment** (like RMSProp): EMA of squared gradients → scale

Plus **bias correction** to account for the zero-initialised moment estimates.

---

### 8.2 Full Algorithm

**Hyperparameters:** $\eta$ (learning rate), $\beta_1$ (1st moment decay), $\beta_2$ (2nd moment decay), $\epsilon$ (numerical stability)

**Initialise:** $\mathbf{m}_0 = \mathbf{0}$, $\mathbf{v}_0 = \mathbf{0}$, $t = 0$

**For each step $t = 1, 2, \ldots$:**

**Step 1 — Compute gradient:**
$$\mathbf{g}_t = \nabla_\theta\mathcal{L}(\theta_{t-1})$$

**Step 2 — Update first moment (momentum):**
$$\mathbf{m}_t = \beta_1\mathbf{m}_{t-1} + (1-\beta_1)\mathbf{g}_t$$

**Step 3 — Update second moment (RMSProp-like):**
$$\mathbf{v}_t = \beta_2\mathbf{v}_{t-1} + (1-\beta_2)\mathbf{g}_t^2$$

**Step 4 — Bias correction:**
$$\hat{\mathbf{m}}_t = \frac{\mathbf{m}_t}{1-\beta_1^t}, \qquad \hat{\mathbf{v}}_t = \frac{\mathbf{v}_t}{1-\beta_2^t}$$

**Step 5 — Update parameters:**
$$\theta_t = \theta_{t-1} - \frac{\eta}{\sqrt{\hat{\mathbf{v}}_t}+\epsilon}\hat{\mathbf{m}}_t$$

---

### 8.3 Why Bias Correction?

At $t=1$ with zero-initialised moments:

$$\mathbf{m}_1 = (1-\beta_1)\mathbf{g}_1$$

Without correction: $\mathbf{m}_1 \approx 0.1\mathbf{g}_1$ for $\beta_1=0.9$ — severely underestimates
the gradient magnitude during early training.

With correction: $\hat{\mathbf{m}}_1 = \mathbf{m}_1/(1-\beta_1) = \mathbf{g}_1$ — correct estimate on step 1.

As $t \to \infty$: $1-\beta_1^t \to 1$, correction vanishes — no effect in later training.

**Derivation:** the moments are EMAs of i.i.d. estimates $\mathbf{g}_k$:

$$\mathbb{E}[\mathbf{m}_t] = (1-\beta_1^t)\mathbb{E}[\mathbf{g}]$$

Dividing by $(1-\beta_1^t)$ gives an unbiased estimate of $\mathbb{E}[\mathbf{g}]$.

---

### 8.4 Effective Step Size

The actual step taken by Adam (combining both corrections):

$$\Delta\theta_t = -\frac{\eta\hat{\mathbf{m}}_t}{\sqrt{\hat{\mathbf{v}}_t}+\epsilon}$$

**Upper bound on step size:**

$$|\Delta\theta_{t,j}| \leq \frac{\eta\,|\hat{m}_{t,j}|}{\sqrt{\hat{v}_{t,j}}+\epsilon} \leq \frac{\eta\cdot|\hat{m}_{t,j}|}{\sqrt{\hat{v}_{t,j}}} = \frac{\eta\cdot|\mathbb{E}[g_j]|}{\sqrt{\mathbb{E}[g_j^2]}} = \eta\cdot\frac{|\mu_j|}{\sigma_j^2}$$

When gradients are noisy ($\mu_j \ll \sigma_j$): step is small.
When gradients are consistent ($\mu_j \approx \sigma_j$): step approaches $\eta$.

**Effective learning rate is bounded:** $|\Delta\theta_{t,j}| \lesssim \eta$ regardless of gradient
magnitude — Adam is **self-normalising**. Parameters always take steps of size $\sim\eta$.

---

### 8.5 Signal-to-Noise Interpretation

$$\frac{\hat{m}_{t,j}}{\sqrt{\hat{v}_{t,j}}} \approx \frac{\mathbb{E}[g_j]}{\sqrt{\mathbb{E}[g_j^2]}} = \frac{\text{mean}}{\text{RMS}} = \text{SNR}_j$$

Adam scales the update by the signal-to-noise ratio of the gradient:
- **High SNR** (consistent gradients, clear direction): take large steps
- **Low SNR** (noisy gradients, uncertain direction): take small steps

This is why Adam often converges faster than SGD — it doesn't waste large steps on noisy parameters.

---

### 8.6 Default Hyperparameters

| Hyperparameter | Default | Range | Effect |
|---|---|---|---|
| $\eta$ (lr) | $10^{-3}$ | $10^{-5}$–$10^{-2}$ | Most sensitive; tune first |
| $\beta_1$ | $0.9$ | $0.8$–$0.99$ | Momentum decay; lower = more reactive |
| $\beta_2$ | $0.999$ | $0.9$–$0.9999$ | Variance decay; lower = faster adaptation |
| $\epsilon$ | $10^{-8}$ | $10^{-10}$–$10^{-6}$ | Prevents div by zero; increase for stability |

$\epsilon$ is rarely tuned, but increasing to $10^{-7}$–$10^{-6}$ can help in some transformer
training scenarios where gradients are very small.

### 8.7 Adam Implementation from Scratch

```python
class Adam:
    """
    Adam — Adaptive Moment Estimation (Kingma & Ba, 2014).

    Combines momentum (1st moment) with adaptive learning rates (2nd moment).
    Bias correction ensures accurate estimates from step 1.

    Update rule:
        m_t = beta1 * m_{t-1} + (1 - beta1) * g_t          # 1st moment
        v_t = beta2 * v_{t-1} + (1 - beta2) * g_t^2         # 2nd moment
        m_hat = m_t / (1 - beta1^t)                          # bias-corrected
        v_hat = v_t / (1 - beta2^t)                          # bias-corrected
        theta = theta - lr / (sqrt(v_hat) + eps) * m_hat     # update

    Args:
        params:       list of parameters with .data and .grad attributes
        lr:           learning rate (alpha)
        betas:        (beta1, beta2) — decay rates for moment estimates
        eps:          numerical stability constant
        weight_decay: L2 regularisation (NOT decoupled — see AdamW for decoupled)
        amsgrad:      use AMSGrad variant (maintains max of v_hat)
    """
    def __init__(self, params: list, lr: float = 1e-3,
                 betas: tuple = (0.9, 0.999), eps: float = 1e-8,
                 weight_decay: float = 0.0, amsgrad: bool = False):
        self.params       = params
        self.lr           = lr
        self.beta1        = betas[0]
        self.beta2        = betas[1]
        self.eps          = eps
        self.weight_decay = weight_decay
        self.amsgrad      = amsgrad

        # Moment buffers — initialised to zero (bias correction handles early steps)
        self.m   = [np.zeros_like(p.data) for p in params]  # 1st moment (mean)
        self.v   = [np.zeros_like(p.data) for p in params]  # 2nd moment (variance)
        self.v_max = [np.zeros_like(p.data) for p in params] if amsgrad else None
        self.t   = 0   # step counter (for bias correction)

    def zero_grad(self):
        for p in self.params:
            p.grad = np.zeros_like(p.data)

    def step(self):
        self.t += 1

        # Precompute bias-correction factors (scalars, not per-parameter)
        bc1 = 1 - self.beta1 ** self.t
        bc2 = 1 - self.beta2 ** self.t

        for i, p in enumerate(self.params):
            if p.grad is None:
                continue

            grad = p.grad.copy()

            # L2 regularisation (adds WD * theta to gradient — NOT decoupled)
            if self.weight_decay != 0:
                grad += self.weight_decay * p.data

            # ── Step 2: Update first moment (EMA of gradients) ────────────────
            self.m[i] = self.beta1 * self.m[i] + (1 - self.beta1) * grad

            # ── Step 3: Update second moment (EMA of squared gradients) ───────
            self.v[i] = self.beta2 * self.v[i] + (1 - self.beta2) * grad ** 2

            # ── Step 4: Bias correction ────────────────────────────────────────
            m_hat = self.m[i] / bc1
            v_hat = self.v[i] / bc2

            # ── AMSGrad: use running max of v_hat ──────────────────────────────
            if self.amsgrad:
                self.v_max[i] = np.maximum(self.v_max[i], v_hat)
                denom = np.sqrt(self.v_max[i]) + self.eps
            else:
                denom = np.sqrt(v_hat) + self.eps

            # ── Step 5: Parameter update ───────────────────────────────────────
            p.data -= self.lr * m_hat / denom

    def get_effective_lr(self) -> float:
        """Current effective learning rate (bias-corrected)."""
        bc1 = 1 - self.beta1 ** self.t
        bc2 = 1 - self.beta2 ** self.t
        return self.lr * np.sqrt(bc2) / bc1

    def __repr__(self):
        return (f"Adam(lr={self.lr}, betas=({self.beta1},{self.beta2}), "
                f"eps={self.eps}, wd={self.weight_decay})")
```

### 8.8 Adam vs SGD — When to Use Which

| Criterion | SGD (+ momentum) | Adam |
|---|---|---|
| **Convergence speed** | Slower (needs careful tuning) | Faster (self-normalising) |
| **Final accuracy** | Often better (if tuned) | Slightly worse in some cases |
| **LR sensitivity** | High — wrong LR → diverge | Lower — effective LR bounded |
| **Sparse gradients** | Poor (same LR for all params) | Excellent (per-param adapt) |
| **Memory** | Low ($O(d)$) | Higher ($O(3d)$ for m,v,t) |
| **Hyperparams to tune** | LR, momentum, schedule | LR, (rarely $\beta_1,\beta_2$) |
| **Recommended for** | CNNs with careful schedule | Transformers, NLP, sparse |

**Rule of thumb:** start with Adam for rapid prototyping. Switch to SGD + cosine schedule
for best final accuracy on image tasks (when you have time to tune).

## 9. AdamW — Decoupled Weight Decay

### 9.1 The Problem with Adam + L2

Standard L2 regularisation adds $\lambda\theta$ to the gradient:

$$\mathbf{g}_t^{\text{L2}} = \mathbf{g}_t + \lambda\theta_{t-1}$$

With Adam, this L2 gradient gets **adapted** by the second moment. Parameters with large
gradient history (large $\hat{v}$) receive a smaller effective weight decay. The regularisation
is not uniform — it depends on the gradient history.

**Consequence:** Adam + L2 $\neq$ Adam + weight decay. The regularisation effect is weakened
and inconsistent.

---

### 9.2 AdamW Solution

**Decouple** the weight decay from the gradient adaptation. Apply the decay directly to the
parameters, bypassing the moment estimates:

**Adam:**
$$\theta_t = \theta_{t-1} - \frac{\eta}{\sqrt{\hat{v}_t}+\epsilon}\hat{m}_t - \frac{\eta\lambda}{\sqrt{\hat{v}_t}+\epsilon}\theta_{t-1}$$

**AdamW:**
$$\theta_t = \theta_{t-1} - \frac{\eta}{\sqrt{\hat{v}_t}+\epsilon}\hat{m}_t - \eta\lambda\theta_{t-1}$$

The decay term $\eta\lambda\theta$ is **not** scaled by $1/\sqrt{\hat{v}_t}$ — it shrinks
all parameters uniformly regardless of gradient history.

---

### 9.3 Implementation

```python
class AdamW:
    """
    AdamW — Adam with Decoupled Weight Decay (Loshchilov & Hutter, 2019).

    Key difference from Adam:
        - Adam:  grad += wd * theta  (then normalise by sqrt(v))
        - AdamW: theta -= lr * wd * theta  SEPARATELY (no normalisation)

    AdamW is the standard for training transformers (BERT, GPT, etc.).

    Update rule:
        m_t   = beta1 * m_{t-1} + (1 - beta1) * g_t
        v_t   = beta2 * v_{t-1} + (1 - beta2) * g_t^2
        m_hat = m_t / (1 - beta1^t)
        v_hat = v_t / (1 - beta2^t)
        theta = theta - lr * m_hat / (sqrt(v_hat) + eps)    # gradient step
                     - lr * weight_decay * theta             # weight decay (DECOUPLED)
    """
    def __init__(self, params: list, lr: float = 1e-3,
                 betas: tuple = (0.9, 0.999), eps: float = 1e-8,
                 weight_decay: float = 0.01):     # higher default than Adam
        self.params = params
        self.lr     = lr
        self.beta1  = betas[0]
        self.beta2  = betas[1]
        self.eps    = eps
        self.wd     = weight_decay

        self.m = [np.zeros_like(p.data) for p in params]
        self.v = [np.zeros_like(p.data) for p in params]
        self.t = 0

    def zero_grad(self):
        for p in self.params:
            p.grad = np.zeros_like(p.data)

    def step(self):
        self.t += 1
        bc1 = 1 - self.beta1 ** self.t
        bc2 = 1 - self.beta2 ** self.t

        for i, p in enumerate(self.params):
            if p.grad is None:
                continue

            grad = p.grad.copy()    # NOTE: no wd added to gradient here

            # Moment updates (pure gradient, no weight decay)
            self.m[i] = self.beta1 * self.m[i] + (1 - self.beta1) * grad
            self.v[i] = self.beta2 * self.v[i] + (1 - self.beta2) * grad ** 2

            # Bias-corrected moments
            m_hat = self.m[i] / bc1
            v_hat = self.v[i] / bc2

            # Gradient step
            p.data -= self.lr * m_hat / (np.sqrt(v_hat) + self.eps)

            # Decoupled weight decay (separate from gradient update)
            p.data -= self.lr * self.wd * p.data

    def __repr__(self):
        return f"AdamW(lr={self.lr}, betas=({self.beta1},{self.beta2}), wd={self.wd})"
```

---

### 9.4 When to Use AdamW

- **Transformers:** BERT, GPT, ViT — AdamW is the default optimizer
- **Any task with L2 regularisation + Adam:** always prefer AdamW over Adam + L2
- **Weight decay values:** AdamW typically uses larger $\lambda$ (e.g., $0.01$–$0.1$) than
  Adam + L2 (e.g., $10^{-4}$) because the decay is uniform and more predictable

## 10. AMSGrad

### 10.1 Convergence Failure of Adam

Adam does not always converge, even on convex problems. Reddi et al. (2018) showed a
counter-example where Adam converges to a suboptimal point because $\hat{v}_t$ can decrease,
causing $\eta_t$ to increase at the wrong time.

**Intuition:** if a parameter rarely receives large gradients, $\hat{v}$ is small → large step.
If the gradient suddenly becomes large, the large step may overshoot. AMSGrad prevents this.

---

### 10.2 AMSGrad Fix

Maintain the **running maximum** of $\hat{v}_t$:

$$\hat{v}_t^{\max} = \max(\hat{v}_{t-1}^{\max}, \hat{v}_t)$$

$$\theta_t = \theta_{t-1} - \frac{\eta}{\sqrt{\hat{v}_t^{\max}}+\epsilon}\hat{m}_t$$

The denominator never decreases — effective LR is non-increasing per parameter.

```python
# AMSGrad is implemented via the amsgrad flag in the Adam class above:
optimizer = Adam(params, lr=1e-3, amsgrad=True)
```

**In practice:** AMSGrad rarely outperforms Adam significantly on real benchmarks. The
counter-example is contrived. Adam's empirical performance is usually fine.

## 11. Learning Rate Schedules

### 11.1 Why Schedule the LR?

**Early training (large LR):** escape poor initialisations, explore the loss landscape broadly.

**Late training (small LR):** converge into the minimum with precision; avoid oscillating around it.

**The fundamental tension:**
- Too large $\eta$: oscillates around the minimum, never converges
- Too small $\eta$: slow convergence, may get stuck in local minima

A schedule starts large (exploration) and decays (exploitation) — best of both worlds.

---

### 11.2 Step Decay

Multiply LR by $\gamma < 1$ every $s$ epochs (or steps):

$$\eta_t = \eta_0 \cdot \gamma^{\lfloor t/s \rfloor}$$

```python
class StepLR:
    def __init__(self, optimizer, step_size: int, gamma: float = 0.1):
        self.opt       = optimizer
        self.step_size = step_size
        self.gamma     = gamma
        self.epoch     = 0

    def step(self):
        self.epoch += 1
        if self.epoch % self.step_size == 0:
            self.opt.lr *= self.gamma

    @property
    def last_lr(self): return self.opt.lr
```

**Drawbacks:** discontinuous jumps in LR can destabilise training briefly after each drop.

---

### 11.3 Exponential Decay

$$\eta_t = \eta_0 \cdot e^{-\lambda t}$$

or equivalently:

$$\eta_t = \eta_0 \cdot \gamma^t, \quad \gamma = e^{-\lambda}$$

Smooth, continuous decay. Too aggressive — LR reaches near-zero quickly.

---

### 11.4 Cosine Annealing

$$\eta_t = \eta_{\min} + \frac{1}{2}(\eta_0 - \eta_{\min})\left(1 + \cos\frac{\pi t}{T}\right)$$

At $t=0$: $\eta_0$ (maximum). At $t=T$: $\eta_{\min}$ (minimum). Smooth transition.

```python
class CosineAnnealingLR:
    """
    Cosine annealing learning rate schedule.
    
    Decays eta from eta_max to eta_min following a cosine curve.
    Smooth — no discontinuous jumps. Standard for ResNet/CNN training.
    """
    def __init__(self, optimizer, T_max: int, eta_min: float = 0.0):
        self.opt     = optimizer
        self.T_max   = T_max
        self.eta_min = eta_min
        self.eta_max = optimizer.lr
        self.t       = 0

    def step(self):
        self.t += 1
        self.opt.lr = (self.eta_min
                       + 0.5 * (self.eta_max - self.eta_min)
                       * (1 + np.cos(np.pi * self.t / self.T_max)))

    @property
    def last_lr(self): return self.opt.lr
```

**Cosine with restarts (SGDR):** reset $t \to 0$ periodically with increasing cycle length.
Each restart explores a new region; the decay helps converge within each cycle.

$$T_{\text{cur}} \to 0: \quad \eta_t \to \eta_0 \quad\text{(restart)}$$

---

### 11.5 Linear Warmup

For large batch sizes or transformers: start LR very small, linearly increase to target.

$$\eta_t = \eta_0 \cdot \frac{t}{T_{\text{warmup}}}, \quad t \leq T_{\text{warmup}}$$

**Why warmup works:**
- At step 1: parameter estimates are random → gradients are unreliable
- Moment estimates (Adam) are biased toward zero (bias correction helps, but variance is high)
- Small initial LR prevents large steps in random directions before the optimizer "warms up"

```python
class WarmupCosine:
    """
    Linear warmup for warmup_steps, then cosine decay to eta_min.
    Standard for transformer training (BERT, GPT).
    """
    def __init__(self, optimizer, warmup_steps: int, total_steps: int,
                 eta_min: float = 0.0):
        self.opt          = optimizer
        self.warmup       = warmup_steps
        self.total        = total_steps
        self.eta_min      = eta_min
        self.eta_max      = optimizer.lr
        self.t            = 0

    def step(self):
        self.t += 1
        if self.t <= self.warmup:
            # Linear warmup
            self.opt.lr = self.eta_max * self.t / self.warmup
        else:
            # Cosine decay after warmup
            progress = (self.t - self.warmup) / (self.total - self.warmup)
            self.opt.lr = (self.eta_min
                           + 0.5 * (self.eta_max - self.eta_min)
                           * (1 + np.cos(np.pi * progress)))
```

---

### 11.6 1-Cycle Policy

Smith (2018): one big cycle — linearly warm up to `max_lr`, then decay via cosine:

```
LR │   ╭────╮
   │  ╱      ╲
   │ ╱          ╲___________
   └──────────────────────── step
     ↑warmup↑   ↑decay phase↑
```

Allows using very high peak learning rates ("super-convergence"):
- Warm up phase: escapes poor initialisations aggressively
- Peak: rapid exploration
- Decay: settle into sharp minimum

```python
class OneCycleLR:
    """
    1-Cycle LR policy (Smith 2018).
    
    Phase 1 (pct_start): linearly warm up from eta_max/div_factor to eta_max
    Phase 2 (1-pct_start): cosine decay to eta_max/final_div_factor
    """
    def __init__(self, optimizer, max_lr: float, total_steps: int,
                 pct_start: float = 0.3, div_factor: float = 25.,
                 final_div_factor: float = 1e4):
        self.opt         = optimizer
        self.max_lr      = max_lr
        self.total_steps = total_steps
        self.warmup_steps= int(pct_start * total_steps)
        self.init_lr     = max_lr / div_factor
        self.min_lr      = max_lr / (div_factor * final_div_factor)
        self.t           = 0

    def step(self):
        self.t += 1
        if self.t <= self.warmup_steps:
            p = self.t / self.warmup_steps
            self.opt.lr = self.init_lr + p * (self.max_lr - self.init_lr)
        else:
            p = (self.t - self.warmup_steps) / (self.total_steps - self.warmup_steps)
            self.opt.lr = (self.min_lr
                           + 0.5 * (self.max_lr - self.min_lr)
                           * (1 + np.cos(np.pi * p)))
```

## 12. Gradient Clipping

### 12.1 Why Gradients Explode

In deep networks, gradients can become very large due to:
- Large activations early in training (poor init)
- Recurrent networks: $\prod_t W$ can grow exponentially
- Sudden sharp loss landscapes (spike in batch loss)

Exploding gradients cause the optimizer to take a massive step, often diverging the loss.

---

### 12.2 Global Gradient Norm Clipping

Compute the **global gradient norm** across all parameters:

$$\|\mathbf{g}\|_2 = \sqrt{\sum_{\theta_i \in \text{all params}}\left(\frac{\partial\mathcal{L}}{\partial\theta_i}\right)^2}$$

If $\|\mathbf{g}\|_2 > \tau$, scale all gradients uniformly to have norm $\tau$:

$$\mathbf{g}_i \leftarrow \mathbf{g}_i \cdot \frac{\tau}{\|\mathbf{g}\|_2} \quad\text{if }\|\mathbf{g}\|_2 > \tau$$

This preserves the **direction** of the gradient — just constrains the step size.

```python
def clip_grad_norm(params: list, max_norm: float, eps: float = 1e-6) -> float:
    """
    Clip gradient of all parameters by global norm.
    
    Args:
        params:   list of parameter objects with .grad attribute
        max_norm: maximum allowed gradient norm (tau)
        eps:      numerical stability
    
    Returns:
        total_norm before clipping (useful for monitoring)
    """
    # ── Compute global L2 norm ─────────────────────────────────────────────
    total_norm = 0.0
    for p in params:
        if p.grad is not None:
            total_norm += np.sum(p.grad ** 2)
    total_norm = np.sqrt(total_norm)

    # ── Scale if norm exceeds max_norm ────────────────────────────────────
    if total_norm > max_norm:
        clip_coeff = max_norm / (total_norm + eps)
        for p in params:
            if p.grad is not None:
                p.grad *= clip_coeff

    return total_norm
```

---

### 12.3 Value Clipping (Alternative)

Clip each gradient component independently:

$$g_i \leftarrow \text{clip}(g_i, -\tau, +\tau)$$

**Drawback:** changes the gradient **direction** (not just magnitude). Not recommended for most
applications — use norm clipping instead.

---

### 12.4 When to Clip

| Use case | Clip? | Threshold |
|---|---|---|
| CNNs (stable) | Optional | 1.0–5.0 |
| RNNs / LSTMs | **Yes** | 1.0–5.0 |
| Transformers | **Yes** | 0.5–1.0 |
| RL / off-policy | **Yes** | 10–40 |
| Loss spikes | Yes | 1.0 |

Monitor the gradient norm during training — a sudden spike often precedes a loss explosion.

## 13. Convergence Theory

### 13.1 Key Definitions

**$L$-smooth:** $\|\nabla\mathcal{L}(\theta) - \nabla\mathcal{L}(\phi)\|_2 \leq L\|\theta - \phi\|_2$
(gradients don't change faster than $L$ per unit distance).

**$\mu$-strongly convex:** $\mathcal{L}(\phi) \geq \mathcal{L}(\theta) + \nabla\mathcal{L}(\theta)^\top(\phi-\theta) + \frac{\mu}{2}\|\phi-\theta\|_2^2$
(curvature at least $\mu$ in every direction).

**Condition number:** $\kappa = L/\mu$ — how ill-conditioned the problem is.

---

### 13.2 Convergence Rates

| Algorithm | Assumption | Rate (iterations to $\epsilon$ accuracy) |
|---|---|---|
| GD | Convex, $L$-smooth | $O(L/\epsilon)$ |
| GD | Strongly convex ($\mu$) | $O(\kappa\log(1/\epsilon))$ |
| Momentum / NAG | Strongly convex | $O(\sqrt{\kappa}\log(1/\epsilon))$ |
| SGD (constant $\eta$) | Convex, noise $\sigma^2$ | $O(\sigma^2/(\mu^2\epsilon))$ noise floor |
| SGD (decaying $\eta$) | Convex | $O(1/\sqrt{T})$ |
| SGD | Non-convex | $O(1/\sqrt{T})$ to stationary point |
| Adam | Non-convex | $O(\log(T)/\sqrt{T})$ in theory |

---

### 13.3 The SGD Noise Floor

With constant LR $\eta$, SGD converges to a **neighbourhood** of the optimum, not the exact minimum:

$$\mathbb{E}[\mathcal{L}(\theta_T)] - \mathcal{L}(\theta^*) \geq \frac{\eta\sigma^2}{2B\mu}$$

**To reach $\epsilon$:** need $\eta \leq \frac{2B\mu\epsilon}{\sigma^2}$ — small LR → slow convergence.

**Optimal strategy:** large LR early (fast descent), then decay (approach minimum precisely).

---

### 13.4 The Sharp vs Flat Minima Debate

**Sharp minimum:** Hessian eigenvalues large — loss increases steeply away from $\theta^*$.
Tends to have higher test loss (generalises poorly).

**Flat minimum:** Hessian eigenvalues small — loss stays low in a large neighbourhood.
Tends to have lower test loss (generalises well).

**Why SGD finds flat minima:**
SGD noise scale $\propto \eta\sigma^2/B$. In a sharp minimum with large curvature $\lambda_{\max}$,
the noise will kick the parameters out — they only stay in flat regions where the curvature
is $O(\eta\sigma^2/B)$.

This is an implicit regularisation effect of stochastic training. Large-batch training finds
sharper minima (less noise) → worse generalisation.

---

### 13.5 Adaptive vs SGD Generalisation Gap

Empirical observation: SGD often generalises better than Adam on image tasks.

**Proposed explanations:**

1. **Sharpness:** Adam adapts to the curvature → may find sharper minima
2. **Implicit L2:** SGD's momentum provides a form of L2 regularisation; Adam's
   gradient normalisation partially cancels the regularisation effect
3. **Effective LR:** Adam's per-parameter LR means some parameters get very large steps,
   potentially overfitting noisy gradient components

**Practical recommendation:**
- Transformers, sparse models, NLP: Adam/AdamW (architecture is designed for it)
- CNNs, ResNets on image data: SGD + cosine often wins when properly tuned

### 14.2 Simple 2-Layer Network

```python
class TwoLayerNet:
    """
    2-layer MLP with ReLU — compatible with our custom optimizers.
    Trains on toy classification data.
    """
    def __init__(self, d_in: int, d_hidden: int, d_out: int):
        # Kaiming init
        std1 = np.sqrt(2.0 / d_in)
        std2 = np.sqrt(2.0 / d_hidden)
        self.W1 = Parameter(np.random.randn(d_hidden, d_in)  * std1)
        self.b1 = Parameter(np.zeros(d_hidden))
        self.W2 = Parameter(np.random.randn(d_out,   d_hidden) * std2)
        self.b2 = Parameter(np.zeros(d_out))
        self.params = [self.W1, self.b1, self.W2, self.b2]

    def forward(self, X: np.ndarray) -> np.ndarray:
        """X: (N, d_in) → logits: (N, d_out)"""
        self.X   = X
        self.z1  = X @ self.W1.data.T + self.b1.data      # (N, d_hidden)
        self.h1  = np.maximum(0, self.z1)                  # ReLU
        self.z2  = self.h1 @ self.W2.data.T + self.b2.data # (N, d_out)
        return self.z2   # raw logits

    def backward(self, d_out: np.ndarray):
        """
        d_out: gradient of loss w.r.t. z2, shape (N, d_out)
        Accumulates gradients into .grad of each parameter.
        """
        N = self.X.shape[0]

        # Output layer
        self.W2.grad += d_out.T @ self.h1 / N    # (d_out, d_hidden)
        self.b2.grad += d_out.mean(axis=0)         # (d_out,)

        # Hidden layer
        d_h1 = d_out @ self.W2.data               # (N, d_hidden)
        d_z1 = d_h1 * (self.z1 > 0)               # ReLU backward
        self.W1.grad += d_z1.T @ self.X / N       # (d_hidden, d_in)
        self.b1.grad += d_z1.mean(axis=0)          # (d_hidden,)


def softmax_cross_entropy(logits: np.ndarray, labels: np.ndarray):
    """
    Numerically stable softmax + cross-entropy.
    Returns (loss scalar, gradient w.r.t. logits).
    """
    N   = logits.shape[0]
    logits = logits - logits.max(axis=1, keepdims=True)   # stability
    exp   = np.exp(logits)
    probs = exp / exp.sum(axis=1, keepdims=True)

    loss  = -np.log(probs[np.arange(N), labels] + 1e-12).mean()

    # Gradient of CE + softmax
    d_logits = probs.copy()
    d_logits[np.arange(N), labels] -= 1
    # Note: gradient already divided by N for mean loss

    return loss, d_logits
```

```python
def train_and_compare(
    X_train: np.ndarray, y_train: np.ndarray,
    X_val:   np.ndarray, y_val:   np.ndarray,
    n_epochs: int = 50, batch_size: int = 64
) -> dict:
    """
    Train with SGD, SGD+Momentum, RMSProp, Adam, AdamW.
    Returns history dict for comparison plots.
    """
    d_in, d_hidden, d_out = X_train.shape[1], 128, len(np.unique(y_train))

    optimizer_configs = {
        'SGD':          lambda p: SGD(p, lr=0.01),
        'SGD+Mom':      lambda p: SGDMomentum(p, lr=0.01, momentum=0.9),
        'SGD+NAG':      lambda p: SGDMomentum(p, lr=0.01, momentum=0.9, nesterov=True),
        'RMSProp':      lambda p: RMSProp(p, lr=0.001, rho=0.99),
        'Adam':         lambda p: Adam(p, lr=0.001, betas=(0.9, 0.999)),
        'AdamW':        lambda p: AdamW(p, lr=0.001, betas=(0.9, 0.999), weight_decay=0.01),
    }

    history = {name: {'train_loss': [], 'val_acc': []}
               for name in optimizer_configs}

    for name, opt_fn in optimizer_configs.items():
        print(f"\nTraining with {name}...")
        net = TwoLayerNet(d_in, d_hidden, d_out)
        opt = opt_fn(net.params)

        for epoch in range(n_epochs):
            # ── Mini-batch training ────────────────────────────────────────────
            perm = np.random.permutation(len(X_train))
            epoch_loss = 0.
            n_batches  = 0

            for start in range(0, len(X_train), batch_size):
                idx    = perm[start:start + batch_size]
                x_b, y_b = X_train[idx], y_train[idx]

                # Forward
                logits = net.forward(x_b)
                loss, d_out = softmax_cross_entropy(logits, y_b)

                # Backward
                for p in net.params: p.zero_grad()
                net.backward(d_out)

                # Gradient clipping
                clip_grad_norm(net.params, max_norm=5.0)

                # Update
                opt.step()

                epoch_loss += loss
                n_batches  += 1

            # ── Validation accuracy ────────────────────────────────────────────
            val_logits = net.forward(X_val)
            val_preds  = val_logits.argmax(axis=1)
            val_acc    = (val_preds == y_val).mean() * 100

            history[name]['train_loss'].append(epoch_loss / n_batches)
            history[name]['val_acc'].append(val_acc)

        final = history[name]['val_acc'][-1]
        print(f"  Final val acc: {final:.1f}%")

    return history
```

```python
import torch
import torch.nn as nn
from torch.optim import Optimizer


class SGDFromScratch(Optimizer):
    """
    SGD reimplemented using PyTorch's Optimizer base class.
    Shows the exact interface that all PyTorch optimizers follow.
    """
    def __init__(self, params, lr: float = 0.01, momentum: float = 0.0,
                 weight_decay: float = 0.0, nesterov: bool = False):
        if lr < 0:
            raise ValueError(f"Invalid lr: {lr}")
        defaults = dict(lr=lr, momentum=momentum,
                        weight_decay=weight_decay, nesterov=nesterov)
        super().__init__(params, defaults)

    @torch.no_grad()
    def step(self, closure=None):
        loss = None
        if closure is not None:
            with torch.enable_grad():
                loss = closure()

        for group in self.param_groups:
            lr           = group['lr']
            momentum     = group['momentum']
            weight_decay = group['weight_decay']
            nesterov     = group['nesterov']

            for p in group['params']:
                if p.grad is None:
                    continue

                grad = p.grad

                # L2 regularisation
                if weight_decay != 0:
                    grad = grad + weight_decay * p

                # Momentum
                if momentum != 0:
                    # Access per-parameter state
                    state = self.state[p]
                    if 'velocity' not in state:
                        state['velocity'] = torch.zeros_like(p)
                    v = state['velocity']
                    v.mul_(momentum).add_(grad)   # v = beta*v + grad

                    if nesterov:
                        grad = grad + momentum * v
                    else:
                        grad = v

                p.add_(grad, alpha=-lr)   # p -= lr * grad

        return loss
```

### 15.2 Adam from Scratch (PyTorch)

```python
class AdamFromScratch(Optimizer):
    """
    Adam implemented from scratch using PyTorch's Optimizer API.
    Matches torch.optim.Adam behavior exactly.
    """
    def __init__(self, params, lr=1e-3, betas=(0.9, 0.999),
                 eps=1e-8, weight_decay=0.0, amsgrad=False):
        defaults = dict(lr=lr, betas=betas, eps=eps,
                        weight_decay=weight_decay, amsgrad=amsgrad)
        super().__init__(params, defaults)

    @torch.no_grad()
    def step(self, closure=None):
        loss = None
        if closure is not None:
            with torch.enable_grad():
                loss = closure()

        for group in self.param_groups:
            lr         = group['lr']
            beta1, beta2 = group['betas']
            eps        = group['eps']
            wd         = group['weight_decay']
            amsgrad    = group['amsgrad']

            for p in group['params']:
                if p.grad is None:
                    continue

                grad = p.grad
                if wd != 0:
                    grad = grad + wd * p   # L2 reg (not decoupled)

                state = self.state[p]

                # Initialise state on first step
                if len(state) == 0:
                    state['step'] = 0
                    state['m']    = torch.zeros_like(p)   # 1st moment
                    state['v']    = torch.zeros_like(p)   # 2nd moment
                    if amsgrad:
                        state['v_max'] = torch.zeros_like(p)

                m, v   = state['m'], state['v']
                state['step'] += 1
                t = state['step']

                # Update moments
                m.mul_(beta1).add_(grad, alpha=1 - beta1)
                v.mul_(beta2).addcmul_(grad, grad, value=1 - beta2)

                # Bias correction
                bc1   = 1 - beta1 ** t
                bc2   = 1 - beta2 ** t
                m_hat = m / bc1
                v_hat = v / bc2

                if amsgrad:
                    v_max = state['v_max']
                    torch.maximum(v_max, v_hat, out=v_max)
                    denom = v_max.sqrt().add_(eps)
                else:
                    denom = v_hat.sqrt().add_(eps)

                p.addcdiv_(m_hat, denom, value=-lr)

        return loss
```

```python
class AdamWFromScratch(Optimizer):
    """
    AdamW: Adam with decoupled weight decay.
    Key line: p -= lr * wd * p  (SEPARATE from gradient step).
    """
    def __init__(self, params, lr=1e-3, betas=(0.9, 0.999),
                 eps=1e-8, weight_decay=0.01):
        defaults = dict(lr=lr, betas=betas, eps=eps, weight_decay=weight_decay)
        super().__init__(params, defaults)

    @torch.no_grad()
    def step(self, closure=None):
        loss = None
        if closure is not None:
            with torch.enable_grad():
                loss = closure()

        for group in self.param_groups:
            lr         = group['lr']
            beta1, beta2 = group['betas']
            eps        = group['eps']
            wd         = group['weight_decay']

            for p in group['params']:
                if p.grad is None:
                    continue

                grad  = p.grad     # pure gradient (no wd added here)
                state = self.state[p]

                if len(state) == 0:
                    state['step'] = 0
                    state['m']    = torch.zeros_like(p)
                    state['v']    = torch.zeros_like(p)

                m, v = state['m'], state['v']
                state['step'] += 1
                t = state['step']

                m.mul_(beta1).add_(grad, alpha=1 - beta1)
                v.mul_(beta2).addcmul_(grad, grad, value=1 - beta2)

                bc1   = 1 - beta1 ** t
                bc2   = 1 - beta2 ** t
                m_hat = m / bc1
                v_hat = v / bc2

                # Gradient step
                p.addcdiv_(m_hat, v_hat.sqrt().add_(eps), value=-lr)

                # ── Decoupled weight decay ─────────────────────────────────────
                p.mul_(1 - lr * wd)    # p -= lr * wd * p

        return loss
```

### 17.1 Common Failure Modes

| Symptom | Likely Cause | Fix |
|---|---|---|
| Loss = NaN from step 1 | LR too large, bad init | Reduce LR by 10×, check init |
| Loss stuck at high value | LR too small | Increase LR; check gradient flow |
| Loss decreases then diverges | LR schedule, batch norm | Add warmup, check BN after training |
| Train loss down, val loss up | Overfitting | Add regularisation, augmentation |
| Train loss oscillates | LR too large | Reduce LR; add gradient clipping |
| Gradient norm spikes | Loss spike in batch | Clip gradients (max_norm=1.0) |
| Dead neurons (ReLU) | Negative bias / large LR | Kaiming init, smaller LR |

```python
class OptimizerMonitor:
    """
    Monitor optimizer state and gradient statistics during training.
    Attach to any optimizer to get real-time diagnostics.
    """

def lr_range_test(model, loader, criterion, device,
                  start_lr=1e-7, end_lr=1.0, n_steps=200):
    """
    Sweep LR log-linearly. Best LR ≈ 10× before loss increases sharply.
    Based on Smith (2017) Cyclical Learning Rates.
    """
    import math
    model_state = {k: v.clone() for k, v in model.state_dict().items()}
    opt   = torch.optim.SGD(model.parameters(), lr=start_lr)
    mult  = (end_lr / start_lr) ** (1 / n_steps)

    lrs, losses   = [], []
    avg_loss = best_loss = float('inf')
    beta = 0.98   # smoothing factor

    model.train()
    data_iter = iter(loader)

    for i in range(n_steps):
        try:
            x, y = next(data_iter)
        except StopIteration:
            data_iter = iter(loader)
            x, y = next(data_iter)

        x, y = x.to(device), y.to(device)
        opt.zero_grad()
        loss = criterion(model(x), y)
        loss.backward()
        opt.step()

        # Exponential smoothing
        avg_loss = beta * avg_loss + (1 - beta) * loss.item()
        if avg_loss == float('inf'):
            avg_loss = loss.item()
        smooth = avg_loss / (1 - beta ** (i + 1))

        lrs.append(opt.param_groups[0]['lr'])
        losses.append(smooth)

        if smooth < best_loss:
            best_loss = smooth
        if smooth > 4 * best_loss:
            break

        for pg in opt.param_groups:
            pg['lr'] *= mult

    # Restore model
    model.load_state_dict(model_state)
    return lrs, losses
```

```python
            x, y = x.to(DEVICE), y.to(DEVICE)
            optimizer.zero_grad()
            out  = model(x)
            loss = criterion(out, y)
            loss.backward()
            nn.utils.clip_grad_norm_(model.parameters(), 1.0)
            optimizer.step()
            epoch_loss += loss.item(); n_batches += 1

        acc = test_acc(model, test_loader)
        hist['loss'].append(epoch_loss / n_batches)
        hist['acc'].append(acc)
        print(f"  {opt_name:<15} Epoch {epoch+1:02d} "
              f"loss={epoch_loss/n_batches:.4f}  acc={acc:.2f}%")

    return hist


# ─────────────────────────────────────────────────────────────────────────────
# Run all optimizers
# ─────────────────────────────────────────────────────────────────────────────

def main():
    train_loader, test_loader = get_loaders(batch_size=128)

    optimizers = {
        'SGD':           lambda p: torch.optim.SGD(p, lr=0.01),

    results = {}

        epochs = range(1, len(hist['loss']) + 1)
```

### 18.2 Convergence on a 2D Quadratic (Visualisation)

```python
"""
Visualise optimizer trajectories on the 2D Rosenbrock function.
Shows how momentum, adaptive LR affect the path to the minimum.
"""

def rosenbrock(x, y, a=1, b=100):
    return (a - x)**2 + b*(y - x**2)**2

def rosenbrock_grad(x, y, a=1, b=100):
    dx = -2*(a - x) - 4*b*x*(y - x**2)
    dy = 2*b*(y - x**2)
    return np.array([dx, dy])

def run_optimizer_2d(opt_class, init=(-1.5, 1.5), n_steps=1000, **kwargs):
    """Run optimizer on Rosenbrock; return trajectory."""
    theta = np.array(list(init), dtype=float)

    class P:
        def __init__(self, data): self.data = data; self.grad = None

    p     = P(theta)
    opt   = opt_class([p], **kwargs)
    traj  = [theta.copy()]

    for _ in range(n_steps):
        p.grad = rosenbrock_grad(*p.data)
        opt.step()
        opt.zero_grad()
        traj.append(p.data.copy())

    return np.array(traj)

# Run each optimizer and plot
configs = {

    'Momentum':  (SGDMomentum, dict(lr=0.001, momentum=0.9)),

    'RMSProp':   (RMSProp,     dict(lr=0.01,  rho=0.99)),
    'Adam':      (Adam,        dict(lr=0.01,  betas=(0.9, 0.999))),
    'AdamW':     (AdamW,       dict(lr=0.01,  betas=(0.9, 0.999), weight_decay=0.01)),
}

import matplotlib.pyplot as plt

for ax, (name, (cls, kw)), color in zip(axes.flat, configs.items(), colors):
    traj = run_optimizer_2d(cls, **kw)
    ax.contour(X, Y, np.log(Z + 1), levels=30, cmap='gray', alpha=0.5)
    ax.plot(traj[:, 0], traj[:, 1], color=color, lw=1.5, alpha=0.8)

plt.tight_layout()
plt.show()
```

## 19. Math Summary

### 19.1 All Update Rules

| Optimizer | Update Rule |
|---|---|
| **SGD** | $\theta \leftarrow \theta - \eta\mathbf{g}$ |
| **SGD + Momentum** | $\mathbf{v} \leftarrow \beta\mathbf{v}+\eta\mathbf{g}$; $\theta \leftarrow \theta-\mathbf{v}$ |
| **Nesterov** | $\mathbf{g} \leftarrow \nabla\mathcal{L}(\theta-\beta\mathbf{v})$; then momentum update |
| **Adagrad** | $G \mathrel{+}= \mathbf{g}^2$; $\theta \leftarrow \theta-\frac{\eta}{\sqrt{G}+\epsilon}\mathbf{g}$ |
| **RMSProp** | $\mathbf{v} \leftarrow \rho\mathbf{v}+(1-\rho)\mathbf{g}^2$; $\theta \leftarrow \theta-\frac{\eta}{\sqrt{\mathbf{v}}+\epsilon}\mathbf{g}$ |
| **Adam** | $\mathbf{m}\leftarrow\beta_1\mathbf{m}+(1-\beta_1)\mathbf{g}$; $\mathbf{v}\leftarrow\beta_2\mathbf{v}+(1-\beta_2)\mathbf{g}^2$; $\theta\leftarrow\theta-\frac{\eta\hat{\mathbf{m}}}{\sqrt{\hat{\mathbf{v}}}+\epsilon}$ |
| **AdamW** | Adam update $+$ $\theta\leftarrow\theta(1-\eta\lambda)$ (decoupled) |
| **AMSGrad** | Adam but $\hat{v}^{\max}_t=\max(\hat{v}^{\max}_{t-1},\hat{v}_t)$; use $\hat{v}^{\max}$ in denom |

---

### 19.2 Key Equations

| Concept | Formula |
|---|---|
| **Loss** | $\mathcal{L}(\theta)=\frac{1}{N}\sum_i\ell(f(\mathbf{x}_i;\theta),y_i)$ |
| **Mini-batch gradient** | $\mathbf{g}_t=\frac{1}{B}\sum_{i\in\mathcal{B}}\nabla\ell_i$ (unbiased: $\mathbb{E}[\mathbf{g}_t]=\nabla\mathcal{L}$) |
| **GD descent guarantee** | $\mathcal{L}(\theta-\eta\mathbf{g})\leq\mathcal{L}(\theta)-\eta(1-\eta L/2)\|\mathbf{g}\|^2$ |
| **GD convergence (SC)** | $\kappa=L/\mu$ steps, rate $\left(\frac{\kappa-1}{\kappa+1}\right)^T$ |
| **Momentum steady state** | $\mathbf{v}_\infty=\eta\mathbf{g}/(1-\beta)$ — amplified by $1/(1-\beta)$ |
| **Momentum convergence** | Rate $\left(\frac{\sqrt{\kappa}-1}{\sqrt{\kappa}+1}\right)^T$ — $\sqrt{\kappa}$ speedup |
| **Adam bias correction** | $\hat{\mathbf{m}}_t=\mathbf{m}_t/(1-\beta_1^t)$; $\hat{\mathbf{v}}_t=\mathbf{v}_t/(1-\beta_2^t)$ |
| **Adam effective LR** | $\|\Delta\theta\|\lesssim\eta$ — bounded regardless of gradient magnitude |
| **Adam SNR** | update $\propto\mathbb{E}[g]/\sqrt{\mathbb{E}[g^2]}=\mu/\sigma$ |
| **SGD noise floor** | $\mathbb{E}[\mathcal{L}]-\mathcal{L}^*\geq\eta\sigma^2/(2B\mu)$ |
| **Linear scaling rule** | $\eta_{\text{new}}=\eta_{\text{base}}\cdot B_{\text{new}}/B_{\text{base}}$ |
| **Cosine LR** | $\eta_t=\eta_{\min}+\frac{1}{2}(\eta_0-\eta_{\min})(1+\cos\pi t/T)$ |
| **Grad norm clip** | $\mathbf{g}\leftarrow\mathbf{g}\cdot\tau/\max(\|\mathbf{g}\|_2,\tau)$ |
| **Weight decay (coupled)** | $\mathbf{g}^{\text{L2}}=\mathbf{g}+\lambda\theta$ then adapt |
| **Weight decay (decoupled)** | $\theta\leftarrow\theta(1-\eta\lambda)$ separately |
| **Adagrad denom** | $G_t=\sum_{k=1}^t\mathbf{g}_k^2$ (accumulates forever) |
| **RMSProp denom** | $\mathbf{v}_t=\rho\mathbf{v}_{t-1}+(1-\rho)\mathbf{g}_t^2$ (EMA — forgets) |

---

### 19.3 Optimizer Family Tree

```
Gradient Descent
        │
        ├── Full-batch GD (exact gradient, slow)
        │
        └── Stochastic GD (mini-batch gradient, fast)
                │
                ├── Vanilla SGD ──────────────────── works, but zig-zags
                │
                ├── + Momentum ──────────────────── damps oscillations
                │       └── + Nesterov ──────────── lookahead gradient
                │
                ├── Adaptive LR methods
                │       │
                │       ├── Adagrad ─────────────── accumulate G_t (dying LR)
                │       │
                │       ├── RMSProp ─────────────── EMA of g^2 (fixes dying LR)
                │       │       └── + Momentum  ── centred RMSProp variant
                │       │
                │       └── Adam ────────────────── momentum + RMSProp + bias corr
                │               ├── AdamW ─────────── decoupled weight decay
                │               ├── AMSGrad ──────── max v_hat for convergence
                │               └── Nadam ─────────── Adam + Nesterov
                │
                └── Schedules (applied to any optimizer)
                        ├── Step decay
                        ├── Cosine annealing
                        ├── Linear warmup + cosine
                        └── 1-Cycle (Smith 2018)
```
