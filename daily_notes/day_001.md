# Einops, Einsum & Tensor Manipulation

> Think of tensors like labeled boxes of data.
> - **Raw manipulation** → "Move the box"
> - **Einops** → "Reorganize how the box is structured, with clear labels"
> - **Einsum** → "Do math across the boxes"

---

## What Is a Tensor?

A tensor is simply a multi-dimensional array — numbers arranged along several axes.

| Dimensions | Name   | Example                      |
|------------|--------|------------------------------|
| 1-D        | Vector | `[features]`                 |
| 2-D        | Matrix | `[rows, cols]`               |
| 3-D+       | Tensor | `[batch, time, embedding]`   |

The trick is that **each dimension has meaning**.

Think of tensors not as boxes, but as **organized shelves in a warehouse**:

```
(batch, sequence, embedding)
  ├── shelf 1 → which example in the batch
  ├── shelf 2 → which token in the sequence
  └── shelf 3 → which feature in the embedding
```

---

## Raw Tensor Manipulation (NumPy / PyTorch)

The **primitive machinery**: `reshape`, `transpose`, `permute`, `view`, `flatten`.

You are manually moving shelves around without semantic clarity.

```python
x = x.permute(0, 2, 1)
# (batch, seq, emb) → (batch, emb, seq)
```

The code tells you *what* is happening but not *why*. You must mentally track axes yourself.

> Like saying: *"Take shelf 2 and swap it with shelf 3."*

**Use raw manipulation for simple, obvious operations:**

```python
x = x.unsqueeze(0)        # Add a dimension
x = x.squeeze(-1)         # Remove a dimension
x = torch.cat([a, b], dim=0)   # Concatenate
x = torch.stack([a, b], dim=0) # Stack into new dim
```

## Einops — Declarative Restructuring

Einops makes tensor reshaping **readable and explicit**.  
Instead of juggling axis numbers, you **name the axes**.

### `rearrange` — Permute / Reshape

```python
from einops import rearrange

# Simple permute — self-documenting
rearrange(x, "batch seq emb -> batch emb seq")

# Split a flat dim into a grid
rearrange(x, "b (h w) c -> b h w c", h=16, w=16)
# Interprets the flattened spatial dim as height × width

# Merge two dims
rearrange(x, "b h w c -> b (h w) c")
```

> `rearrange` performs **no arithmetic** — only reorganization.

### `reduce` — Aggregation Over Named Axes

```python
from einops import reduce

reduce(x, "b seq emb -> b emb", "mean")  # Pool over sequence
reduce(x, "b h w c -> b c", "max")       # Spatial global max pool
```

### `repeat` — Tile / Broadcast

```python
from einops import repeat

repeat(x, "b emb -> b n emb", n=10)  # Expand to n copies
```

> **Einops is a language for describing tensor structure.**

## Einsum — Tensor Algebra

Einsum performs **mathematical operations** across dimensions.  
It comes from *Einstein summation notation* used in physics.

```python
torch.einsum("ij,jk->ik", A, B)   # Matrix multiplication
```

### How to Read It

```
  i j
  j k
  ---
  i k
```

- The **shared index** `j` is summed over (contracted).
- **Rule:** If an index appears on the left but not the right → it is summed over.
- **Rule:** If an index appears on both sides → it is kept.

### Common Einsum Patterns

```python
# Matrix multiplication
torch.einsum("ij,jk->ik", A, B)

# Batched matrix multiplication
torch.einsum("bij,bjk->bik", A, B)

# Batched dot products (per-sample)
torch.einsum("bd,bd->b", A, B)

# Outer product
torch.einsum("i,j->ij", a, b)

# Gram / covariance matrix
torch.einsum("id,jd->ij", A, A)

# Attention scores
torch.einsum("bhid,bhjd->bhij", Q, K)

# Attention output (weighted aggregation)
torch.einsum("bhij,bhjd->bhid", scores, V)
```

## Which Tool to Use?

```
Does it involve multiply + sum (math)?
  YES → EINSUM
    · Matrix multiply, batched matmul
    · Dot products (per-sample, all-pairs)
    · Attention scores and weighted aggregation
    · Gram / covariance matrix
    · Bilinear forms, tensor contractions

Is it a structural reshape / permute?
  Complex (split/merge dims, 3+ axes) → EINOPS rearrange
  Aggregation (pool/reduce)           → EINOPS reduce
  Expansion (tile/broadcast)          → EINOPS repeat

Is it simple and obvious?
  YES → RAW (torch.cat, unsqueeze, squeeze, view)
```

## Operation Reference Table

| Operation                  | Tool             | Example                          |
|----------------------------|------------------|----------------------------------|
| Add / remove dim           | Raw              | `.unsqueeze(0)`, `.squeeze(-1)`  |
| Concatenate tensors        | Raw              | `torch.cat([a, b], dim=0)`       |
| Stack tensors              | Raw              | `torch.stack([a, b], dim=0)`     |
| Complex permute (3+ dims)  | Einops rearrange | `"b s e -> b e s"`               |
| Split dim into two         | Einops rearrange | `"b (h d) -> b h d"`             |
| Merge two dims             | Einops rearrange | `"b h d -> b (h d)"`             |
| Pool over named axes       | Einops reduce    | `"b s e -> b e", "mean"`         |
| Tile / broadcast           | Einops repeat    | `"b e -> b n e"`                 |
| Matrix multiply            | Einsum           | `"ij,jk->ik"`                    |
| Batched dot products       | Einsum           | `"bd,bd->b"`                     |
| Attention scores           | Einsum           | `"bhid,bhjd->bhij"`              |
| Attention output           | Einsum           | `"bhij,bhjd->bhid"`              |
| Outer product              | Einsum           | `"i,j->ij"`                      |
| Gram matrix                | Einsum           | `"id,jd->ij"`                    |

---

## Summary

| Tool    | Purpose             | Does Math? | Names Axes? | Best For                     |
|---------|---------------------|------------|-------------|------------------------------|
| Raw     | Primitive ops       | No         | No          | Simple, obvious reshapes     |
| Einops  | Declarative reshape | No*        | Yes         | Complex restructuring        |
| Einsum  | Tensor algebra      | Yes        | Yes         | Multiply + sum across dims   |

> *Einops `reduce` does aggregate math (mean, sum, max), but no elementwise multiply.
