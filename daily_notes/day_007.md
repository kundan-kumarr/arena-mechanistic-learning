## 1. The MNIST Dataset

### 1.1 Overview

MNIST (Modified National Institute of Standards and Technology) is the canonical benchmark for
handwritten digit classification — the "Hello, World" of deep learning.

| Attribute | Value |
|---|---|
| Creator | LeCun, Cortes, Burges (1998) |
| Training samples | 60,000 |
| Test samples | 10,000 |
| Image size | 28 × 28 pixels |
| Channels | 1 (grayscale) |
| Classes | 10 (digits 0–9) |
| Pixel range | 0–255 (uint8) → normalised |
| File format | IDX binary (`.gz`) |

---

### 1.2 Dataset Statistics

**Per-class distribution (training set):**

| Digit | Count | Fraction |
|---|---|---|
| 0 | 5,923 | 9.87% |
| 1 | 6,742 | 11.24% |
| 2 | 5,958 | 9.93% |
| 3 | 6,131 | 10.22% |
| 4 | 5,842 | 9.74% |
| 5 | 5,421 | 9.04% |
| 6 | 5,918 | 9.86% |
| 7 | 6,265 | 10.44% |
| 8 | 5,851 | 9.75% |
| 9 | 5,949 | 9.92% |

Near-uniform distribution — no severe class imbalance. Class 5 is slightly underrepresented.

**Global statistics (training set):**

$$\mu = 0.1307, \qquad \sigma = 0.3081$$

These are the standard MNIST normalisation constants used across all implementations.

---

### 1.3 Why MNIST?

MNIST is useful for learning because:

- **Small enough** to train a strong model in minutes on CPU
- **Simple enough** that the main challenge is coding, not tuning
- **Rich enough** to showcase MLPs, CNNs, regularisation, and visualisation
- **Well-understood** — expected accuracy ranges are known, so bugs are obvious

**When MNIST is not enough:** MNIST is too easy for modern architectures. A simple CNN achieves
~99.5% — leaving little room for improvement. For harder benchmarks use: CIFAR-10, Fashion-MNIST,
KMNIST, or EMNIST.

---

### 1.4 IDX File Format

MNIST is stored in a custom binary format:

```
Training images: train-images-idx3-ubyte.gz
  Magic number:    0x00000803  (4 bytes)
  Number of items: 60000       (4 bytes, big-endian int32)
  Rows:            28          (4 bytes)
  Cols:            28          (4 bytes)
  Pixel data:      60000 × 784 bytes

Training labels: train-labels-idx1-ubyte.gz
  Magic number:    0x00000801
  Number of items: 60000
  Label data:      60000 × 1 byte (0–9)
```

PyTorch's `torchvision.datasets.MNIST` handles the download and parsing automatically.

## 2. Preprocessing & Normalisation

### 2.1 The Normalisation Transform

Raw pixels are in $[0, 255]$ (uint8). We first convert to float $[0, 1]$ via `ToTensor()`,
then standardise to zero mean and unit variance:

$$x_{\text{norm}} = \frac{x - \mu}{\sigma} = \frac{x - 0.1307}{0.3081}$$

**Why normalise?**

Without normalisation:
- Inputs have mean $\approx 33$ and std $\approx 78$
- Gradients w.r.t. weights are proportional to input magnitude → very large gradients
- Loss surface is ill-conditioned: learning rates must be tiny

After normalisation:
- Mean $\approx 0$, std $\approx 1$
- Gradients are well-scaled → larger learning rates are stable
- Consistent scale across layers → BatchNorm more effective

**After normalisation, pixel values span approximately $[-0.42, 2.82]$:**

$$x_{\min} = (0 - 0.1307)/0.3081 \approx -0.424$$
$$x_{\max} = (1 - 0.1307)/0.3081 \approx 2.821$$

---

### 2.2 Loading MNIST with PyTorch

```python
import torch
from torch.utils.data import DataLoader, random_split
import torchvision
import torchvision.transforms as T


MNIST_MEAN = (0.1307,)   # tuple — one value per channel
MNIST_STD  = (0.3081,)


def get_mnist_loaders(
    data_dir:    str  = './data',
    batch_size:  int  = 64,
    num_workers: int  = 2,
    val_split:   float= 0.1,     # fraction of train set used for validation
    augment:     bool = False,   # enable training augmentation
) -> tuple:
    """
    Return (train_loader, val_loader, test_loader).

    Val set is carved out of the training set (stratified by default via shuffle).
    Test set is the official 10k MNIST test split — never touch during training.
    """

    # ── Transforms ────────────────────────────────────────────────────────────
    base_tf = T.Compose([
        T.ToTensor(),                           # HWC uint8 → CHW float32 in [0,1]
        T.Normalize(MNIST_MEAN, MNIST_STD),     # standardise
    ])

    train_tf = T.Compose([
        T.RandomAffine(                         # data augmentation
            degrees=10,                         # rotate ±10°
            translate=(0.1, 0.1),               # shift ±10% of image size
            scale=(0.9, 1.1),                   # zoom ±10%
        ),
        T.ToTensor(),
        T.Normalize(MNIST_MEAN, MNIST_STD),
    ]) if augment else base_tf

    # ── Datasets ──────────────────────────────────────────────────────────────
    full_train = torchvision.datasets.MNIST(
        root=data_dir, train=True,
        transform=train_tf, download=True
    )
    test_dataset = torchvision.datasets.MNIST(
        root=data_dir, train=False,
        transform=base_tf, download=True         # test always uses base_tf
    )

    # ── Train / Val split ─────────────────────────────────────────────────────
    n_val   = int(len(full_train) * val_split)   # e.g. 6,000
    n_train = len(full_train) - n_val            # e.g. 54,000
    train_dataset, val_dataset = random_split(
        full_train, [n_train, n_val],
        generator=torch.Generator().manual_seed(42)
    )

    # ── DataLoaders ───────────────────────────────────────────────────────────
    common_kw = dict(num_workers=num_workers, pin_memory=True)
    train_loader = DataLoader(train_dataset, batch_size,
                              shuffle=True, drop_last=True, **common_kw)
    val_loader   = DataLoader(val_dataset,   batch_size,
                              shuffle=False,              **common_kw)
    test_loader  = DataLoader(test_dataset,  batch_size,
                              shuffle=False,              **common_kw)

    print(f"Train: {len(train_dataset):,}  Val: {len(val_dataset):,}"
          f"  Test: {len(test_dataset):,}")
    print(f"Batches/epoch — Train: {len(train_loader)}"
          f"  Val: {len(val_loader)}  Test: {len(test_loader)}")

    return train_loader, val_loader, test_loader
```

## 3. Model 1 — Logistic Regression Baseline

### 3.1 What It Is

Logistic regression on flattened pixels: a single linear layer followed by softmax.

$$\hat{\mathbf{y}} = \text{softmax}(W\mathbf{x} + \mathbf{b})$$

where $\mathbf{x} \in \mathbb{R}^{784}$ (flattened $28\times28$), $W \in \mathbb{R}^{10\times784}$, $\mathbf{b} \in \mathbb{R}^{10}$.

**Parameters:** $10 \times 784 + 10 = 7{,}850$

**Expected accuracy:** ~92% — treats every pixel independently, no spatial structure.

---

### 3.2 The Math

**Forward pass:**

$$\mathbf{z} = W\mathbf{x} + \mathbf{b} \in \mathbb{R}^{10}$$

$$\hat{p}_k = \frac{e^{z_k}}{\sum_{j=0}^{9}e^{z_j}} \quad\text{(softmax)}$$

**Loss — cross-entropy:**

$$\mathcal{L} = -\log\hat{p}_c = -z_c + \log\sum_{j=0}^{9}e^{z_j}$$

where $c$ is the true class. This is the negative log-likelihood under the categorical distribution.

**Gradient w.r.t. logits:**

$$\frac{\partial\mathcal{L}}{\partial z_k} = \hat{p}_k - \mathbf{1}[k=c]$$

**Gradient w.r.t. weights:**

$$\frac{\partial\mathcal{L}}{\partial W} = (\hat{\mathbf{p}} - \mathbf{y})\mathbf{x}^\top \in \mathbb{R}^{10\times784}$$

The weight matrix $W$ can be interpreted as 10 "prototype" images — one per class. The row
$W[c]$ learns to look like the average digit $c$.

---

### 3.3 Implementation

```python
import torch
import torch.nn as nn


class LogisticRegression(nn.Module):
    """
    Single linear layer — the simplest possible MNIST classifier.
    Establishes the accuracy floor: ~92%.
    """
    def __init__(self, input_dim: int = 784, num_classes: int = 10):
        super().__init__()
        self.flatten = nn.Flatten()               # (N,1,28,28) → (N,784)
        self.linear  = nn.Linear(input_dim, num_classes)

        # Zero-init weights (random init works too — LR is convex)
        nn.init.zeros_(self.linear.weight)
        nn.init.zeros_(self.linear.bias)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.linear(self.flatten(x))       # returns logits (no softmax)
        # Note: nn.CrossEntropyLoss applies softmax internally


# ── Quick test ────────────────────────────────────────────────────────────────
model = LogisticRegression()
x     = torch.randn(8, 1, 28, 28)                # batch of 8 images
out   = model(x)
print(f"Logits shape: {out.shape}")               # (8, 10)
print(f"Parameters:   {sum(p.numel() for p in model.parameters()):,}")  # 7,850
```

## 4. Model 2 — Multi-Layer Perceptron (MLP)

### 4.1 Architecture

A stack of fully-connected layers with non-linear activations:

```
Input (N, 1, 28, 28)
    │
    Flatten → (N, 784)
    │
    Linear(784 → 512)  + BatchNorm1d(512) + ReLU + Dropout(0.2)
    │
    Linear(512 → 256)  + BatchNorm1d(256) + ReLU + Dropout(0.2)
    │
    Linear(256 → 128)  + BatchNorm1d(128) + ReLU + Dropout(0.2)
    │
    Linear(128 → 10)   [logits]
    │
    Output (N, 10)
```

**Parameter count:**

| Layer | Weights | Bias | BN | Total |
|---|---|---|---|---|
| Linear 784→512 | 401,408 | 512 | 1,024 | 402,944 |
| Linear 512→256 | 131,072 | 256 | 512 | 131,840 |
| Linear 256→128 | 32,768 | 128 | 256 | 33,152 |
| Linear 128→10 | 1,280 | 10 | — | 1,290 |
| **Total** | | | | **569,226** |

**Expected accuracy:** ~98.5% with proper regularisation.

---

### 4.2 Why Each Component?

**BatchNorm1d after linear:**

For layer input $\mathbf{z} \in \mathbb{R}^{B\times d}$:

$$\mu_j = \frac{1}{B}\sum_{n=1}^B z_{n,j}, \qquad \sigma_j^2 = \frac{1}{B}\sum_{n=1}^B(z_{n,j}-\mu_j)^2$$

$$\hat{z}_{n,j} = \frac{z_{n,j}-\mu_j}{\sqrt{\sigma_j^2+\epsilon}}, \qquad y_{n,j} = \gamma_j\hat{z}_{n,j}+\beta_j$$

BN normalises each feature dimension across the batch. For 1D (fully-connected) layers,
`nn.BatchNorm1d` normalises over the batch dimension for each feature independently.

**Dropout (regularisation):**

$$\tilde{h}_j = \begin{cases}h_j/(1-p) & \text{w.p. } 1-p \\ 0 & \text{w.p. } p\end{cases}$$

During training only — at test time, all neurons are active (no scaling needed because of
inverted dropout convention).

---

### 4.3 Implementation

```python
class MLP(nn.Module):
    """
    Multi-Layer Perceptron for MNIST.

    Features:
        - Configurable hidden dimensions
        - BatchNorm1d after each linear layer (except last)
        - Dropout for regularisation
        - Kaiming weight initialisation
    """
    def __init__(
        self,
        hidden_dims: list  = [512, 256, 128],
        dropout:     float = 0.2,
        num_classes: int   = 10,
    ):
        super().__init__()
        self.flatten = nn.Flatten()

        # ── Build hidden layers dynamically ───────────────────────────────────
        layers    = []
        in_dim    = 784
        for h_dim in hidden_dims:
            layers += [
                nn.Linear(in_dim, h_dim),
                nn.BatchNorm1d(h_dim),
                nn.ReLU(inplace=True),
                nn.Dropout(p=dropout),
            ]
            in_dim = h_dim

        # ── Output layer (no BN, no activation — raw logits) ─────────────────
        layers.append(nn.Linear(in_dim, num_classes))

        self.network = nn.Sequential(*layers)
        self._init_weights()

    def _init_weights(self):
        """Kaiming normal for linear layers (designed for ReLU)."""
        for m in self.modules():
            if isinstance(m, nn.Linear):
                nn.init.kaiming_normal_(m.weight, nonlinearity='relu')
                nn.init.zeros_(m.bias)
            elif isinstance(m, nn.BatchNorm1d):
                nn.init.ones_(m.weight)
                nn.init.zeros_(m.bias)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.network(self.flatten(x))


# ── Inspect ───────────────────────────────────────────────────────────────────
mlp     = MLP([512, 256, 128], dropout=0.2)
n_params= sum(p.numel() for p in mlp.parameters())
x       = torch.randn(16, 1, 28, 28)
print(f"Output shape: {mlp(x).shape}")    # (16, 10)
print(f"Parameters:   {n_params:,}")      # ~569K
```

---

### 4.4 MLP Forward Pass — Full Math

For a 2-hidden-layer MLP, input $\mathbf{x}\in\mathbb{R}^{784}$:

**Layer 1:**
$$\mathbf{z}^{(1)} = W^{(1)}\mathbf{x} + \mathbf{b}^{(1)} \in \mathbb{R}^{512}$$
$$\hat{\mathbf{z}}^{(1)} = \text{BN}(\mathbf{z}^{(1)}) = \gamma^{(1)}\odot\frac{\mathbf{z}^{(1)}-\boldsymbol{\mu}}{\boldsymbol{\sigma}+\epsilon} + \boldsymbol{\beta}^{(1)}$$
$$\mathbf{h}^{(1)} = \text{ReLU}(\hat{\mathbf{z}}^{(1)})$$
$$\tilde{\mathbf{h}}^{(1)} = \text{Dropout}(\mathbf{h}^{(1)}, p=0.2)$$

**Layer 2:**
$$\mathbf{z}^{(2)} = W^{(2)}\tilde{\mathbf{h}}^{(1)} + \mathbf{b}^{(2)} \in \mathbb{R}^{256}$$
$$\text{(same BN, ReLU, Dropout pipeline)}$$

**Output:**
$$\mathbf{z}^{(3)} = W^{(3)}\tilde{\mathbf{h}}^{(2)} + \mathbf{b}^{(3)} \in \mathbb{R}^{10} \quad\text{(logits)}$$
$$\hat{\mathbf{p}} = \text{softmax}(\mathbf{z}^{(3)}), \qquad \hat{c} = \arg\max_k\hat{p}_k$$

---

### 4.5 MLP Limitations

| Limitation | Explanation |
|---|---|
| Ignores spatial structure | Pixel $(0,0)$ and pixel $(27,27)$ treated identically |
| No translation equivariance | Shifting the digit breaks the representation entirely |
| Large parameter count | 784 inputs × wide hidden layers = millions of params |
| Quadratic scaling | Doubling image size → 4× more parameters in first layer |

These limitations motivate convolutional models.

## 5. Model 3 — Convolutional Neural Network (CNN)

### 5.1 Architecture Design

```
Input (N, 1, 28, 28)
    │
    ┌── Block 1 ──────────────────────────────────────────┐
    │  Conv2d(1→32, 3×3, pad=1)  → (N, 32, 28, 28)        │
    │  BatchNorm2d(32) + ReLU                              │
    │  Conv2d(32→32, 3×3, pad=1) → (N, 32, 28, 28)        │
    │  BatchNorm2d(32) + ReLU                              │
    │  MaxPool2d(2×2, stride=2)  → (N, 32, 14, 14)        │
    │  Dropout2d(0.25)                                     │
    └──────────────────────────────────────────────────────┘
    │
    ┌── Block 2 ──────────────────────────────────────────┐
    │  Conv2d(32→64, 3×3, pad=1) → (N, 64, 14, 14)       │
    │  BatchNorm2d(64) + ReLU                              │
    │  Conv2d(64→64, 3×3, pad=1) → (N, 64, 14, 14)       │
    │  BatchNorm2d(64) + ReLU                              │
    │  MaxPool2d(2×2, stride=2)  → (N, 64,  7,  7)        │
    │  Dropout2d(0.25)                                     │
    └──────────────────────────────────────────────────────┘
    │
    GlobalAvgPool → (N, 64)
    │
    Linear(64 → 128) + ReLU + Dropout(0.5)
    │
    Linear(128 → 10)  [logits]
    │
    Output (N, 10)
```

**Parameter count:**

| Layer | Params |
|---|---|
| Conv 1→32 (3×3) | $32\times1\times9+32 = 320$ |
| Conv 32→32 (3×3) | $32\times32\times9+32 = 9{,}248$ |
| Conv 32→64 (3×3) | $64\times32\times9+64 = 18{,}496$ |
| Conv 64→64 (3×3) | $64\times64\times9+64 = 36{,}928$ |
| BN layers (×4) | $4\times(64+64+128+128) = \approx 1{,}440$ |
| Linear 64→128 | $64\times128+128 = 8{,}320$ |
| Linear 128→10 | $128\times10+10 = 1{,}290$ |
| **Total** | **~76,000** |

8× fewer parameters than the MLP, yet higher accuracy — spatial structure matters.

---

### 5.2 Why Two Conv Layers Per Block?

One $3\times3$ conv has receptive field 3. Two stacked $3\times3$ convs have receptive field 5 —
same as one $5\times5$ — but use fewer parameters and have one extra non-linearity:

| Option | RF | Params (for 32 channels) | Non-linearities |
|---|---|---|---|
| One $5\times5$ | 5 | $32\times32\times25 = 25{,}600$ | 1 |
| Two $3\times3$ | 5 | $2\times32\times32\times9 = 18{,}432$ | 2 |

Two $3\times3$ convs win on both parameter count and expressiveness.

---

### 5.3 Spatial Dimension Arithmetic

For MNIST (28×28 input):

| Stage | Operation | Output Shape |
|---|---|---|
| Input | — | $(N,1,28,28)$ |
| Block 1 Conv ×2 | $k=3, P=1, S=1$ | $(N,32,28,28)$ |
| Block 1 MaxPool | $k=2, S=2$ | $(N,32,14,14)$ |
| Block 2 Conv ×2 | $k=3, P=1, S=1$ | $(N,64,14,14)$ |
| Block 2 MaxPool | $k=2, S=2$ | $(N,64,7,7)$ |
| GlobalAvgPool | — | $(N,64,1,1)$ |
| Flatten | — | $(N,64)$ |
| Linear ×2 | — | $(N,10)$ |

---

### 5.4 Implementation

```python
class ConvBlock(nn.Module):
    """Two 3×3 convs + BN + ReLU + MaxPool + Dropout2d."""
    def __init__(self, in_ch: int, out_ch: int, dropout: float = 0.25):
        super().__init__()
        self.block = nn.Sequential(
            nn.Conv2d(in_ch,  out_ch, kernel_size=3, padding=1, bias=False),
            nn.BatchNorm2d(out_ch),
            nn.ReLU(inplace=True),

            nn.Conv2d(out_ch, out_ch, kernel_size=3, padding=1, bias=False),
            nn.BatchNorm2d(out_ch),
            nn.ReLU(inplace=True),

            nn.MaxPool2d(kernel_size=2, stride=2),
            nn.Dropout2d(p=dropout),              # spatial dropout (whole channels)
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.block(x)


class MNISTConvNet(nn.Module):
    """
    Two-block CNN for MNIST.

    Expected accuracy: ~99.3% without augmentation.
                      ~99.5% with augmentation.
    """
    def __init__(self, dropout_conv: float = 0.25, dropout_fc: float = 0.5):
        super().__init__()

        # ── Convolutional feature extractor ───────────────────────────────────
        self.features = nn.Sequential(
            ConvBlock(1,  32, dropout=dropout_conv),   # 28→14
            ConvBlock(32, 64, dropout=dropout_conv),   # 14→7
        )

        # ── Classifier head ───────────────────────────────────────────────────
        self.pool       = nn.AdaptiveAvgPool2d((1, 1))  # 7→1  (GlobalAvgPool)
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(64, 128),
            nn.ReLU(inplace=True),
            nn.Dropout(p=dropout_fc),
            nn.Linear(128, 10),
        )

        self._init_weights()

    def _init_weights(self):
        for m in self.modules():
            if isinstance(m, nn.Conv2d):
                nn.init.kaiming_normal_(m.weight, mode='fan_out',
                                        nonlinearity='relu')
            elif isinstance(m, nn.BatchNorm2d):
                nn.init.ones_(m.weight)
                nn.init.zeros_(m.bias)
            elif isinstance(m, nn.Linear):
                nn.init.kaiming_normal_(m.weight, nonlinearity='relu')
                nn.init.zeros_(m.bias)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        x = self.features(x)
        x = self.pool(x)
        return self.classifier(x)

    def get_features(self, x: torch.Tensor) -> torch.Tensor:
        """Extract feature embeddings (before classifier) — useful for visualisation."""
        x = self.features(x)
        x = self.pool(x)
        return torch.flatten(x, 1)   # (N, 64)


# ── Shape trace ───────────────────────────────────────────────────────────────
cnn = MNISTConvNet()
x   = torch.randn(4, 1, 28, 28)

print("Shape trace:")
h = x
h = cnn.features[0](h); print(f"  After Block 1: {h.shape}")  # (4, 32, 14, 14)
h = cnn.features[1](h); print(f"  After Block 2: {h.shape}")  # (4, 64, 7, 7)
h = cnn.pool(h);        print(f"  After GAP:     {h.shape}")  # (4, 64, 1, 1)
h = cnn.classifier(h);  print(f"  After FC:      {h.shape}")  # (4, 10)

print(f"\nTotal parameters: {n_params:,}")
```

### 5.5 What the Conv Filters Learn

**First layer filters** (1→32 channels): learn low-level detectors — horizontal edges, vertical
edges, diagonal edges, local blobs.

**Second layer filters** (32→64 channels): combine first-layer patterns — detect curve fragments,
strokes, corners. Larger effective receptive field (5×5 after two $3\times3$ convs plus pooling).

**Why convolutions outperform MLPs on MNIST:**

- Translation equivariance: a stroke detected at position $(i,j)$ is equally valid at $(i+1,j)$
- Weight sharing: same filter reused across all $14\times14$ positions after first pool → far fewer parameters per concept learned
- Hierarchical features: later layers build on early-layer detections

## 6. Model 4 — ResNet-style MNIST

### 6.1 Architecture

A lightweight ResNet with residual connections, scaled for MNIST's small images:

```
Input (N, 1, 28, 28)
    │
    Stem: Conv2d(1→16, 3×3, pad=1) + BN + ReLU  → (N, 16, 28, 28)
    │
    ResBlock(16→16)   → (N, 16, 28, 28)
    ResBlock(16→16)   → (N, 16, 28, 28)
    │
    ResBlock(16→32, stride=2)  → (N, 32, 14, 14)   [downsample + proj shortcut]
    ResBlock(32→32)            → (N, 32, 14, 14)
    │
    ResBlock(32→64, stride=2)  → (N, 64, 7, 7)
    ResBlock(64→64)            → (N, 64, 7, 7)
    │
    GlobalAvgPool  → (N, 64)
    │
    Linear(64→10)  [logits]
```

**Total parameters:** ~180K — more than the CNN, but residual connections give better gradient flow.

**Expected accuracy:** ~99.5%

### 6.2 Implementation

```python
class ResBlock(nn.Module):
    """
    BasicBlock residual unit for MNIST.

    Identity shortcut if dimensions match.
    Projection shortcut (1×1 conv) if channels or stride differ.
    """
    def __init__(self, in_ch: int, out_ch: int, stride: int = 1):
        super().__init__()

        # ── Main path ─────────────────────────────────────────────────────────
        self.path = nn.Sequential(
            nn.Conv2d(in_ch, out_ch, 3, stride=stride, padding=1, bias=False),
            nn.BatchNorm2d(out_ch),
            nn.ReLU(inplace=True),
            nn.Conv2d(out_ch, out_ch, 3, stride=1, padding=1, bias=False),
            nn.BatchNorm2d(out_ch),
        )

        # ── Shortcut ──────────────────────────────────────────────────────────
        self.shortcut = nn.Identity()
        if stride != 1 or in_ch != out_ch:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_ch, out_ch, 1, stride=stride, bias=False),
                nn.BatchNorm2d(out_ch),
            )

        self.relu = nn.ReLU(inplace=True)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.relu(self.path(x) + self.shortcut(x))


class TinyResNet(nn.Module):
    """Lightweight ResNet for MNIST — residuals on 28×28 images."""
    def __init__(self, num_classes: int = 10):
        super().__init__()

        self.stem = nn.Sequential(
            nn.Conv2d(1, 16, 3, stride=1, padding=1, bias=False),
            nn.BatchNorm2d(16),
            nn.ReLU(inplace=True),
        )

        self.layer1 = nn.Sequential(ResBlock(16, 16), ResBlock(16, 16))
        self.layer2 = nn.Sequential(ResBlock(16, 32, stride=2), ResBlock(32, 32))
        self.layer3 = nn.Sequential(ResBlock(32, 64, stride=2), ResBlock(64, 64))

        self.pool = nn.AdaptiveAvgPool2d(1)
        self.fc   = nn.Linear(64, num_classes)

        for m in self.modules():
            if isinstance(m, nn.Conv2d):
                nn.init.kaiming_normal_(m.weight, mode='fan_out',
                                        nonlinearity='relu')
            elif isinstance(m, nn.BatchNorm2d):
                nn.init.ones_(m.weight); nn.init.zeros_(m.bias)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        x = self.stem(x)
        x = self.layer1(x)
        x = self.layer2(x)
        x = self.layer3(x)
        x = torch.flatten(self.pool(x), 1)
        return self.fc(x)
```

## 7. Loss Functions

### 7.1 Cross-Entropy Loss (Standard)

$$\mathcal{L}_{\text{CE}} = -\sum_{k=0}^{9}y_k\log\hat{p}_k = -\log\hat{p}_c$$

where $y_k = \mathbf{1}[k=c]$ is one-hot and $\hat{p}_k = \text{softmax}(\mathbf{z})_k$.

**Numerically stable implementation** — log-sum-exp trick:

$$\log\hat{p}_c = z_c - \log\sum_{k=0}^{9}e^{z_k} = z_c - \left(m + \log\sum_{k=0}^{9}e^{z_k-m}\right), \quad m=\max_k z_k$$

PyTorch's `nn.CrossEntropyLoss` combines `LogSoftmax` + `NLLLoss` with this stability built in.

---

### 7.2 Negative Log-Likelihood (NLL)

If you apply `nn.LogSoftmax` in the forward pass:

```python
log_probs = F.log_softmax(logits, dim=1)
loss      = F.nll_loss(log_probs, targets)
```

Mathematically identical to cross-entropy:

$$\text{NLL} = -\log\hat{p}_c = \mathcal{L}_{\text{CE}}$$

---

### 7.3 Label Smoothing

Replace hard one-hot labels with smoothed distribution:

$$\tilde{y}_k = (1-\varepsilon)\cdot\mathbf{1}[k=c] + \frac{\varepsilon}{K}$$

For $\varepsilon=0.1$, $K=10$: true class gets $0.91$, others get $0.01$.

**Effect:** prevents overconfident predictions (logits → $\infty$ for true class).
Small but consistent improvement on MNIST (~0.05% accuracy gain).

```python
criterion_ls = nn.CrossEntropyLoss(label_smoothing=0.1)
```

**Modified loss:**

$$\mathcal{L}_{\text{LS}} = (1-\varepsilon)\mathcal{L}_{\text{CE}} + \frac{\varepsilon}{K}\sum_{k=0}^{K-1}\mathcal{L}_k$$

---

### 7.4 Focal Loss (for Hard Examples)

Down-weights easy examples (confident correct predictions), focuses training on hard ones:

$$\mathcal{L}_{\text{focal}} = -(1-\hat{p}_c)^\gamma\log\hat{p}_c$$

For $\gamma=0$: standard cross-entropy. For $\gamma=2$: near-zero loss when $\hat{p}_c>0.9$.

```python
class FocalLoss(nn.Module):
    def __init__(self, gamma: float = 2.0):
        super().__init__()
        self.gamma = gamma

    def forward(self, logits: torch.Tensor, targets: torch.Tensor) -> torch.Tensor:
        log_probs = F.log_softmax(logits, dim=1)     # (N, K)
        probs     = log_probs.exp()                   # (N, K)

        # Gather true class probabilities
        true_probs = probs.gather(1, targets.unsqueeze(1)).squeeze(1)  # (N,)
        true_log_p = log_probs.gather(1, targets.unsqueeze(1)).squeeze(1)

        # Focal weight: (1 - p_c)^gamma
        focal_weight = (1 - true_probs) ** self.gamma

        return -(focal_weight * true_log_p).mean()
```

On MNIST, focal loss is rarely needed (the dataset is clean and well-balanced). Useful for
imbalanced or noisy digit datasets.

## 8. Optimizers & Schedulers

### 8.1 SGD with Momentum

$$\mathbf{v}_t = \beta\mathbf{v}_{t-1} + (1-\beta)\mathbf{g}_t, \qquad \theta_t = \theta_{t-1} - \eta\mathbf{v}_t$$

```python
optimizer = torch.optim.SGD(
    model.parameters(),
    lr=0.1,           # high LR enabled by BN
    momentum=0.9,
    weight_decay=1e-4,
    nesterov=True,
)
```

**Nesterov momentum** computes the gradient at the *lookahead* position:

$$\mathbf{g}_t = \nabla_\theta\mathcal{L}(\theta_{t-1} - \beta\mathbf{v}_{t-1})$$

Slightly better than standard momentum — "looks ahead" before accumulating velocity.

---

### 8.2 Adam

$$\mathbf{m}_t = \beta_1\mathbf{m}_{t-1}+(1-\beta_1)\mathbf{g}_t \quad\text{(1st moment)}$$
$$\mathbf{v}_t = \beta_2\mathbf{v}_{t-1}+(1-\beta_2)\mathbf{g}_t^2 \quad\text{(2nd moment)}$$
$$\hat{\mathbf{m}}_t = \mathbf{m}_t/(1-\beta_1^t), \quad \hat{\mathbf{v}}_t = \mathbf{v}_t/(1-\beta_2^t) \quad\text{(bias correction)}$$
$$\theta_t = \theta_{t-1} - \eta\hat{\mathbf{m}}_t/(\sqrt{\hat{\mathbf{v}}_t}+\epsilon)$$

```python
optimizer = torch.optim.Adam(
    model.parameters(),
    lr=1e-3,
    betas=(0.9, 0.999),
    eps=1e-8,
    weight_decay=1e-4,
)
```

Adam converges faster than SGD on MNIST but SGD often achieves better final accuracy with
careful tuning. For MNIST, both achieve ~99.3–99.5%.

### 8.3 Learning Rate Schedules

**Step Decay:**

$$\eta_t = \eta_0 \cdot \gamma^{\lfloor t/\text{step}\rfloor}$$


**Cosine Annealing:**

$$\eta_t = \eta_{\min} + \frac{1}{2}(\eta_0 - \eta_{\min})\left(1 + \cos\frac{\pi t}{T}\right)$$

```python

    optimizer, T_max=20, eta_min=1e-6

```

**ReduceLROnPlateau** (adaptive — monitors validation loss):


### 8.4 1-Cycle Policy (Smith 2018)

One big cycle: linearly warm up LR to `max_lr`, then decay via cosine to `min_lr/div_factor`:

```python
scheduler = torch.optim.lr_scheduler.OneCycleLR(

    div_factor=25,        # start LR = max_lr / 25 = 0.004
    final_div_factor=1e4, # end LR = max_lr / 250000
)
# Must call scheduler.step() after every BATCH (not epoch)
```

1-cycle often achieves the same accuracy in half the epochs.

## 9. Weight Initialisation

### 9.1 Why It Matters for MNIST

For a 3-layer MLP with 512 units and ReLU:

- **All zeros:** all hidden units identical → symmetric gradients → no learning
- **$\mathcal{N}(0, 0.01)$:** activations collapse to near-zero after a few layers
- **Xavier:** $\sigma^2 = 2/(d_{\text{in}}+d_{\text{out}})$ — designed for tanh, not ReLU
- **Kaiming/He:** $\sigma^2 = 2/d_{\text{in}}$ — designed for ReLU, accounts for the half-zeroing

---

### 9.2 Kaiming Derivation

For a ReLU layer with $d_{\text{in}}$ inputs, the pre-activation variance:

$$\text{Var}(z_j) = d_{\text{in}} \cdot \sigma_w^2 \cdot \text{Var}(h_{\text{prev}})$$

ReLU zeroes half the inputs on average:

$$\text{Var}(h_j) = \frac{1}{2}\text{Var}(z_j) = \frac{d_{\text{in}}}{2}\sigma_w^2\cdot\text{Var}(h_{\text{prev}})$$

Setting $\text{Var}(h_j) = \text{Var}(h_{\text{prev}})$ (constant variance across layers):

$$\boxed{\sigma_w^2 = \frac{2}{d_{\text{in}}}}$$

```python
# PyTorch: fan_in mode (default) for hidden layers
nn.init.kaiming_normal_(weight, mode='fan_in', nonlinearity='relu')

# fan_out mode for output layers (preserves variance in backward pass)
nn.init.kaiming_normal_(weight, mode='fan_out', nonlinearity='relu')
```

## 10. Regularisation

### 10.1 L2 Weight Decay

$$\mathcal{L}_{\text{reg}} = \mathcal{L}_{\text{CE}} + \frac{\lambda}{2}\sum_{W}\|W\|_F^2$$

Gradient: $\partial\mathcal{L}_{\text{reg}}/\partial W = \partial\mathcal{L}/\partial W + \lambda W$

In the update: $W \leftarrow W(1-\eta\lambda) - \eta\partial\mathcal{L}/\partial W$ — shrinks weights toward zero.

**Typical values for MNIST:**

| $\lambda$ | Effect |
|---|---|
| $0$ | No regularisation — can overfit on noise |
| $10^{-5}$ | Very mild — recommended for CNN |
| $10^{-4}$ | Standard — good for MLP |
| $10^{-3}$ | Strong — may underfit |

---

### 10.2 Dropout

**Dropout2d** (for convolutional layers): zeroes entire feature maps (channels), not individual units:

$$\tilde{F}[c] = \begin{cases}F[c]/(1-p) & \text{w.p. }1-p \\ \mathbf{0} & \text{w.p. }p\end{cases}$$

Zeroing entire channels forces redundancy — other channels must carry the same information.

**Standard Dropout** (for dense layers): zeroes individual units:

$$\tilde{h}_j = \begin{cases}h_j/(1-p) & \text{w.p. }1-p \\ 0 & \text{w.p. }p\end{cases}$$

The scaling $1/(1-p)$ (inverted dropout) ensures $\mathbb{E}[\tilde{h}_j] = h_j$ — no adjustment needed at test time.

---

### 10.3 Data Augmentation for MNIST

MNIST is simple enough that aggressive augmentation can hurt (distorted digits become ambiguous).
Safe augmentations:

```python
train_tf = T.Compose([
```

**Augmentations to AVOID on MNIST:**

| Augmentation | Reason to avoid |

| RandomHorizontalFlip | Flips 2↔2 (ok) but 3,4,7 look wrong |
| RandomVerticalFlip | 6↔9 confusion |
| Large rotation (>15°) | 6↔9, 1↔7 confusion |
| Strong color jitter | Grayscale — no color to jitter |
| Large cutout | May erase the entire digit |

### 10.4 Early Stopping

```python
class EarlyStopping:
    """
    Stop training when validation loss stops improving.

    Args:
        patience: epochs to wait before stopping
        delta:    minimum improvement to reset patience counter
        path:     where to save the best model
    """
    def __init__(self, patience: int = 7, delta: float = 1e-4,
                 path: str = 'best_model.pt'):
        self.patience = patience
        self.delta    = delta
        self.path     = path
        self.counter  = 0
        self.best_loss = float('inf')
        self.early_stop = False

    def __call__(self, val_loss: float, model: nn.Module) -> bool:
        if val_loss < self.best_loss - self.delta:
            self.best_loss = val_loss
            self.counter   = 0
            torch.save(model.state_dict(), self.path)
            return False   # continue training
        else:
            self.counter += 1
            print(f"  EarlyStopping: {self.counter}/{self.patience}")
            if self.counter >= self.patience:
                self.early_stop = True
                return True    # stop training
            return False
```

## 11. Full Training Pipeline

### 11.1 Training Step

```python
def train_epoch(
    model:     nn.Module,
    loader:    DataLoader,
    optimizer: torch.optim.Optimizer,
    criterion: nn.Module,
    device:    torch.device,
    scheduler = None,          # optional per-batch scheduler (OneCycleLR)
) -> tuple:
    """
    One full training epoch.
    Returns: (avg_loss, accuracy_pct, avg_grad_norm)
    """
    model.train()
    total_loss = correct = total = 0
    grad_norms = []

    for x, y in loader:
        x, y = x.to(device, non_blocking=True), y.to(device, non_blocking=True)

        # ── Forward ────────────────────────────────────────────────────────
        logits = model(x)
        loss   = criterion(logits, y)

        # ── Backward ───────────────────────────────────────────────────────
        optimizer.zero_grad(set_to_none=True)
        loss.backward()

        # ── Gradient clipping ──────────────────────────────────────────────
        grad_norm = nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
        grad_norms.append(grad_norm.item())

        # ── Update ─────────────────────────────────────────────────────────
        optimizer.step()
        if scheduler is not None:
            scheduler.step()   # per-batch (OneCycleLR)

        # ── Metrics ────────────────────────────────────────────────────────
        with torch.no_grad():
            total_loss += loss.item() * x.size(0)
            correct    += logits.argmax(1).eq(y).sum().item()
            total      += x.size(0)

    return total_loss/total, 100.*correct/total, sum(grad_norms)/len(grad_norms)


@torch.no_grad()
def evaluate(
    model:     nn.Module,
    loader:    DataLoader,
    criterion: nn.Module,
    device:    torch.device,
) -> tuple:
    """Evaluation (validation or test). Returns: (avg_loss, accuracy_pct)"""
    model.eval()
    total_loss = correct = total = 0

    for x, y in loader:
        x, y = x.to(device, non_blocking=True), y.to(device, non_blocking=True)
        logits = model(x)
        total_loss += criterion(logits, y).item() * x.size(0)
        correct    += logits.argmax(1).eq(y).sum().item()
        total      += x.size(0)

    return total_loss/total, 100.*correct/total
```

### 11.2 Full Training Loop

```python
from dataclasses import dataclass


@dataclass
class Config:
    # Model
    model:         str   = 'cnn'       # 'logreg' | 'mlp' | 'cnn' | 'resnet'
    hidden_dims:   list  = None        # MLP hidden dims [512, 256, 128]

    # Data
    batch_size:    int   = 128
    augment:       bool  = True
    val_split:     float = 0.1

    # Training
    epochs:        int   = 20
    lr:            float = 1e-3
    weight_decay:  float = 1e-4
    label_smooth:  float = 0.0
    optimizer:     str   = 'adam'      # 'sgd' | 'adam'
    scheduler:     str   = 'cosine'    # 'none' | 'step' | 'cosine' | 'plateau'

    # Misc
    device:        str   = 'cuda'
    seed:          int   = 42
    patience:      int   = 7           # early stopping patience


def run_training(cfg: Config):
    import random, numpy as np

    # ── Seed ──────────────────────────────────────────────────────────────────
    torch.manual_seed(cfg.seed)
    np.random.seed(cfg.seed)
    random.seed(cfg.seed)
    device = torch.device(cfg.device if torch.cuda.is_available() else 'cpu')
    print(f"Device: {device}")

    # ── Data ──────────────────────────────────────────────────────────────────
    train_loader, val_loader, test_loader = get_mnist_loaders(
        batch_size=cfg.batch_size, augment=cfg.augment, val_split=cfg.val_split
    )

    # ── Model ─────────────────────────────────────────────────────────────────
    models = {
        'logreg': LogisticRegression,
        'mlp':    lambda: MLP(cfg.hidden_dims or [512,256,128], cfg.dropout),
        'cnn':    lambda: MNISTConvNet(cfg.dropout, cfg.dropout * 2),
        'resnet': TinyResNet,
    }
    model = models[cfg.model]().to(device)
    n_params = sum(p.numel() for p in model.parameters() if p.requires_grad)
    print(f"Model: {cfg.model} | Params: {n_params:,}")

    # ── Loss ──────────────────────────────────────────────────────────────────
    criterion = nn.CrossEntropyLoss(label_smoothing=cfg.label_smooth)

    # ── Optimizer ─────────────────────────────────────────────────────────────
    if cfg.optimizer == 'sgd':
        optimizer = torch.optim.SGD(model.parameters(), lr=cfg.lr,
                                    momentum=0.9, weight_decay=cfg.weight_decay,
                                    nesterov=True)
    else:
        optimizer = torch.optim.Adam(model.parameters(), lr=cfg.lr,
                                     weight_decay=cfg.weight_decay)

    # ── Scheduler ─────────────────────────────────────────────────────────────
    if   cfg.scheduler == 'cosine':
        scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
            optimizer, T_max=cfg.epochs, eta_min=1e-6)
        per_batch_sched = None
    elif cfg.scheduler == 'step':
        scheduler = torch.optim.lr_scheduler.StepLR(
            optimizer, step_size=5, gamma=0.5)
        per_batch_sched = None
    elif cfg.scheduler == 'plateau':
        scheduler = torch.optim.lr_scheduler.ReduceLROnPlateau(
            optimizer, patience=3, factor=0.5, min_lr=1e-6)
        per_batch_sched = None
    else:
        scheduler = per_batch_sched = None

    # ── Early stopping ────────────────────────────────────────────────────────
    early_stop = EarlyStopping(patience=cfg.patience)

    # ── Training ──────────────────────────────────────────────────────────────
    history = {'tr_loss':[],'tr_acc':[],'val_loss':[],'val_acc':[],'lr':[]}
    best_test_acc = 0.

    print(f"\n{'Ep':>3} {'LR':>8} {'TrLoss':>8} {'TrAcc':>7} "
          f"{'VLoss':>8} {'VAcc':>7} {'GNorm':>7}")
    print("─" * 60)

    for epoch in range(1, cfg.epochs + 1):
        tr_loss, tr_acc, gnorm = train_epoch(
            model, train_loader, optimizer, criterion, device, per_batch_sched)
        val_loss, val_acc      = evaluate(model, val_loader, criterion, device)

        # ── Scheduler step ────────────────────────────────────────────────────
        cur_lr = optimizer.param_groups[0]['lr']
        if scheduler is not None and per_batch_sched is None:
            if cfg.scheduler == 'plateau':
                scheduler.step(val_loss)
            else:
                scheduler.step()

        # ── Log ───────────────────────────────────────────────────────────────
        for k, v in zip(['tr_loss','tr_acc','val_loss','val_acc','lr'],
                        [tr_loss, tr_acc, val_loss, val_acc, cur_lr]):
            history[k].append(v)

        print(f"{epoch:>3d} {cur_lr:>8.5f} {tr_loss:>8.4f} {tr_acc:>6.2f}% "
              f"{val_loss:>8.4f} {val_acc:>6.2f}% {gnorm:>7.4f}")

        # ── Early stopping ────────────────────────────────────────────────────
        if early_stop(val_loss, model):
            print("Early stopping triggered.")
            break

    # ── Final test ────────────────────────────────────────────────────────────
    model.load_state_dict(torch.load('best_model.pt'))
    test_loss, test_acc = evaluate(model, test_loader, criterion, device)
    print(f"\nTest accuracy: {test_acc:.2f}%  |  Test loss: {test_loss:.4f}")
    return history, model
```

## 12. Evaluation & Metrics

### 12.1 Accuracy

$$\text{Accuracy} = \frac{\text{correct predictions}}{\text{total predictions}} = \frac{1}{N}\sum_{i=1}^N\mathbf{1}[\hat{c}_i = c_i]$$

For MNIST, accuracy is the primary metric (balanced classes, symmetric misclassification costs).

---

### 12.2 Confusion Matrix

$(i,j)$-th entry = number of test samples with true label $i$ predicted as $j$:

$$C_{ij} = \sum_{n=1}^N\mathbf{1}[c_n=i]\cdot\mathbf{1}[\hat{c}_n=j]$$

Diagonal = correct. Off-diagonal = errors.

**Common confusions in MNIST:**

| True | Predicted | Reason |
|---|---|---|
| 4 | 9 | Long vertical stroke, closed top |
| 3 | 8 | Similar curves, top and bottom |
| 5 | 6 | Both have a curve at the bottom |
| 7 | 1 | Both are mostly vertical |
| 2 | 7 | Diagonal stroke similarity |

```python
def plot_confusion_matrix(model, loader, device, class_names=None):
    import matplotlib.pyplot as plt
    import numpy as np

    if class_names is None:
        class_names = [str(i) for i in range(10)]

    model.eval()
    cm = torch.zeros(10, 10, dtype=torch.long)
    with torch.no_grad():
        for x, y in loader:
            preds = model(x.to(device)).argmax(1).cpu()
            for t, p in zip(y, preds):
                cm[t, p] += 1

    cm_np = cm.numpy()
    cm_norm = cm_np.astype(float) / cm_np.sum(axis=1, keepdims=True)

    fig, axes = plt.subplots(1, 2, figsize=(14, 5))
    for ax, data, title, fmt in [
        (axes[0], cm_np,   'Count',      'd'),
        (axes[1], cm_norm, 'Normalised', '.2f'),
    ]:
        im = ax.imshow(data, cmap='Blues')
        ax.set_xticks(range(10)); ax.set_yticks(range(10))
        ax.set_xticklabels(class_names); ax.set_yticklabels(class_names)
        ax.set_xlabel('Predicted'); ax.set_ylabel('True')
        ax.set_title(title)
        plt.colorbar(im, ax=ax)
        for i in range(10):
            for j in range(10):
                color = 'white' if data[i,j] > data.max()*0.6 else 'black'
                ax.text(j, i, format(data[i,j], fmt),
                        ha='center', va='center', color=color, fontsize=7)

    plt.suptitle('MNIST Confusion Matrix', fontsize=13)
    plt.tight_layout()
    plt.show()
    return cm
```

### 12.3 Per-Class Precision, Recall, F1

$$\text{Precision}_c = \frac{C_{cc}}{\sum_i C_{ic}}, \qquad \text{Recall}_c = \frac{C_{cc}}{\sum_j C_{cj}}, \qquad F1_c = \frac{2\cdot P_c\cdot R_c}{P_c+R_c}$$

```python
def classification_report(cm: torch.Tensor) -> None:
    """Print per-class precision, recall, F1 from confusion matrix."""
    cm = cm.float()
    precision = torch.diag(cm) / cm.sum(0).clamp(min=1)
    recall    = torch.diag(cm) / cm.sum(1).clamp(min=1)
    f1        = 2 * precision * recall / (precision + recall).clamp(min=1e-8)
    support   = cm.sum(1).long()

    print(f"\n{'Digit':>6} {'Prec':>8} {'Rec':>8} {'F1':>8} {'Support':>9}")
    print("─" * 45)
    for d in range(10):
        print(f"{d:>6d} {precision[d]:>8.4f} {recall[d]:>8.4f}"
              f" {f1[d]:>8.4f} {support[d]:>9d}")
    print(f"\n{'Macro avg':>6}  {precision.mean():>7.4f}  {recall.mean():>7.4f}"
          f"  {f1.mean():>7.4f} {support.sum():>9d}")
```

### 12.4 Calibration

A well-calibrated model: when it predicts class $c$ with probability $p$, it should be correct
$p\times100\%$ of the time.

**Expected Calibration Error (ECE):**

$$\text{ECE} = \sum_{m=1}^M\frac{|B_m|}{N}\left|\text{acc}(B_m) - \text{conf}(B_m)\right|$$

where $B_m$ are confidence bins (e.g., $[0, 0.1), [0.1, 0.2), \ldots, [0.9, 1.0]$).

```python
def compute_ece(model, loader, device, n_bins=10):
    """Compute Expected Calibration Error."""
    model.eval()
    confidences, corrects = [], []

    with torch.no_grad():
        for x, y in loader:
            probs    = torch.softmax(model(x.to(device)), dim=1).cpu()
            conf, pred = probs.max(1)
            confidences.extend(conf.tolist())
            corrects.extend(pred.eq(y).tolist())

    confidences = torch.tensor(confidences)
    corrects    = torch.tensor(corrects).float()
    bins        = torch.linspace(0, 1, n_bins + 1)
    ece         = 0.

    for i in range(n_bins):
        mask = (confidences > bins[i]) & (confidences <= bins[i+1])
        if mask.sum() > 0:
            bin_acc  = corrects[mask].mean().item()
            bin_conf = confidences[mask].mean().item()
            ece     += mask.float().mean().item() * abs(bin_acc - bin_conf)

    print(f"ECE: {ece:.4f}  (lower is better; 0 = perfect calibration)")
    return ece
```

```python
def plot_feature_maps(model, image: torch.Tensor, max_maps=32):
```

### 13.5 Saliency Maps (Gradient-Based)

Shows which pixels most influence a particular prediction:

$$\text{Saliency}(\mathbf{x})_i = \left|\frac{\partial\hat{p}_c}{\partial x_i}\right|$$

```python
def saliency_map(model, image: torch.Tensor, target_class: int = None):
    """
    Compute vanilla saliency map via input gradients.
    Highlights pixels that most affect the predicted class score.
    """
    import matplotlib.pyplot as plt

    model.eval()
    x = image.unsqueeze(0).requires_grad_(True)

    # Forward
    logits = model(x)
    c      = logits.argmax(1).item() if target_class is None else target_class

    # Backward — gradient of class score w.r.t. input pixels
    model.zero_grad()
    logits[0, c].backward()

    saliency = x.grad.data.abs().squeeze().cpu()
    saliency = (saliency - saliency.min()) / (saliency.max() + 1e-8)

    fig, axes = plt.subplots(1, 3, figsize=(9, 3))
    # Original
    img_disp = (image * 0.3081 + 0.1307).clamp(0,1).squeeze().cpu()
    axes[0].imshow(img_disp, cmap='gray'); axes[0].set_title('Input')
    # Saliency
    axes[1].imshow(saliency, cmap='hot'); axes[1].set_title(f'Saliency (class {c})')
    # Overlay
    axes[2].imshow(img_disp, cmap='gray', alpha=0.5)
    axes[2].imshow(saliency, cmap='hot',  alpha=0.5)
    axes[2].set_title('Overlay')
    for ax in axes: ax.axis('off')
    plt.tight_layout(); plt.show()
```

```python
def find_hard_examples(model, loader, device, n=25):
    """
    Find the n most-confidently wrong predictions.
    These reveal systematic failure modes.
    """
    model.eval()
    errors = []   # (loss, image, true_label, pred_label, confidence)

    with torch.no_grad():
        for x, y in loader:
            x_dev = x.to(device)
            probs  = torch.softmax(model(x_dev), dim=1).cpu()
            preds  = probs.argmax(1)
            confs, _ = probs.max(1)

            wrong = (preds != y).nonzero(as_tuple=True)[0]
            for idx in wrong:
                errors.append((
                    confs[idx].item(),   # confidence in wrong prediction
                    x[idx].clone(),
                    y[idx].item(),
                    preds[idx].item(),
                ))

    # Sort by confidence (most confident wrong = hardest)
    errors.sort(key=lambda e: -e[0])
    errors = errors[:n]

    # Display
    import matplotlib.pyplot as plt
    cols = 5; rows = (n + cols - 1) // cols
    fig, axes = plt.subplots(rows, cols, figsize=(cols * 2, rows * 2))
    mean, std = 0.1307, 0.3081

    for i, ax in enumerate(axes.flat):
        if i < len(errors):
            conf, img, true, pred = errors[i]
            img_disp = (img * std + mean).clamp(0,1).squeeze()
            ax.imshow(img_disp, cmap='gray')
            ax.set_title(f'True:{true} Pred:{pred}\nConf:{conf:.2f}',
                         fontsize=8, color='red')
        ax.axis('off')

    plt.suptitle(f'Top {n} Confident Errors', fontsize=12)
    plt.tight_layout()
    plt.show()
    return errors
```

```python
def analyse_uncertainty(model, loader, device, n_examples=8):
    """
    Show predicted probability bar charts for error cases.
    High entropy = uncertain. Low entropy = overconfident wrong prediction.
    """
    import matplotlib.pyplot as plt
```

### 15.2 Key Hyperparameter Effects on MNIST

| Hyperparameter | Typical range | Effect |
|---|---|---|
| Learning rate | $10^{-4}$–$10^{-1}$ | Most sensitive; too high → diverge, too low → slow |
| Batch size | 32–512 | Larger = faster/epoch but may need lower LR |
| Dropout (conv) | 0.1–0.4 | Prevents overfitting; too high → underfits |

| Weight decay | $10^{-5}$–$10^{-3}$ | Mild regularisation; MNIST rarely needs strong |
| Hidden dims | [256–1024] | Diminishing returns above 512 for MNIST |

```python
"""
MNIST Classifier — Complete self-contained script.
Trains a CNN to ~99.3% test accuracy in ~10 epochs on CPU (5 min),
or ~2 min on GPU.

Usage: python mnist_classifier.py
"""

import torch
import torch.nn as nn
import torch.nn.functional as F
import torchvision
import torchvision.transforms as T
from torch.utils.data import DataLoader, random_split


# ─────────────────────────────────────────────────────────────────────────────
# Constants
# ─────────────────────────────────────────────────────────────────────────────

MNIST_MEAN  = (0.1307,)
MNIST_STD   = (0.3081,)
NUM_CLASSES = 10
DEVICE      = torch.device('cuda' if torch.cuda.is_available() else 'cpu')


# ─────────────────────────────────────────────────────────────────────────────
# Data
# ─────────────────────────────────────────────────────────────────────────────

def get_loaders(batch_size=128, augment=True):
    train_tf = T.Compose([
        T.RandomAffine(degrees=10, translate=(0.1, 0.1), scale=(0.9, 1.1)),
        T.ToTensor(),
        T.Normalize(MNIST_MEAN, MNIST_STD),
    ]) if augment else T.Compose([T.ToTensor(), T.Normalize(MNIST_MEAN, MNIST_STD)])

    test_tf = T.Compose([T.ToTensor(), T.Normalize(MNIST_MEAN, MNIST_STD)])

    full  = torchvision.datasets.MNIST('./data', True,  train_tf, download=True)
    test  = torchvision.datasets.MNIST('./data', False, test_tf,  download=True)
    n_val = 5000
    train, val = random_split(full, [len(full)-n_val, n_val],
                              generator=torch.Generator().manual_seed(42))

    kw = dict(num_workers=2, pin_memory=True)
    return (DataLoader(train, batch_size, shuffle=True,  drop_last=True, **kw),
            DataLoader(val,   batch_size, shuffle=False,               **kw),
            DataLoader(test,  batch_size, shuffle=False,               **kw))


# ─────────────────────────────────────────────────────────────────────────────
# Model
# ─────────────────────────────────────────────────────────────────────────────

class ConvNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.features = nn.Sequential(
            # Block 1: 28×28 → 14×14
            nn.Conv2d(1, 32, 3, padding=1, bias=False), nn.BatchNorm2d(32), nn.ReLU(True),
            nn.Conv2d(32, 32, 3, padding=1, bias=False), nn.BatchNorm2d(32), nn.ReLU(True),
            nn.MaxPool2d(2, 2), nn.Dropout2d(0.25),

            # Block 2: 14×14 → 7×7
            nn.Conv2d(32, 64, 3, padding=1, bias=False), nn.BatchNorm2d(64), nn.ReLU(True),
            nn.Conv2d(64, 64, 3, padding=1, bias=False), nn.BatchNorm2d(64), nn.ReLU(True),
            nn.MaxPool2d(2, 2), nn.Dropout2d(0.25),
        )
        self.head = nn.Sequential(
            nn.AdaptiveAvgPool2d(1), nn.Flatten(),
            nn.Linear(64, 128), nn.ReLU(True), nn.Dropout(0.5),
            nn.Linear(128, NUM_CLASSES),
        )
        self._init()

    def _init(self):
        for m in self.modules():
            if isinstance(m, nn.Conv2d):
                nn.init.kaiming_normal_(m.weight, mode='fan_out', nonlinearity='relu')
            elif isinstance(m, nn.BatchNorm2d):
                nn.init.ones_(m.weight); nn.init.zeros_(m.bias)
            elif isinstance(m, nn.Linear):
                nn.init.kaiming_normal_(m.weight); nn.init.zeros_(m.bias)

    def forward(self, x):
        return self.head(self.features(x))


# ─────────────────────────────────────────────────────────────────────────────
# Training
# ─────────────────────────────────────────────────────────────────────────────

def train_one_epoch(model, loader, opt, crit):
    model.train()
    loss_sum = correct = total = 0
    for x, y in loader:
        x, y = x.to(DEVICE), y.to(DEVICE)
        opt.zero_grad(set_to_none=True)
        out  = model(x)
        loss = crit(out, y)
        loss.backward()
        nn.utils.clip_grad_norm_(model.parameters(), 1.0)
        opt.step()
        loss_sum += loss.item() * x.size(0)
        correct  += out.argmax(1).eq(y).sum().item()
        total    += x.size(0)
    return loss_sum/total, 100.*correct/total


@torch.no_grad()
def eval_loop(model, loader, crit):
    model.eval()
    loss_sum = correct = total = 0
    for x, y in loader:
        x, y = x.to(DEVICE), y.to(DEVICE)
        out  = model(x)
        loss_sum += crit(out, y).item() * x.size(0)
        correct  += out.argmax(1).eq(y).sum().item()
        total    += x.size(0)
    return loss_sum/total, 100.*correct/total


# ─────────────────────────────────────────────────────────────────────────────
# Main
# ─────────────────────────────────────────────────────────────────────────────

def main():
    EPOCHS = 20
    BATCH  = 128
    LR     = 1e-3

    torch.manual_seed(42)
    print(f"Device: {DEVICE}")

    train_loader, val_loader, test_loader = get_loaders(BATCH, augment=True)
    model     = ConvNet().to(DEVICE)
    criterion = nn.CrossEntropyLoss(label_smoothing=0.1)
    optimizer = torch.optim.Adam(model.parameters(), lr=LR, weight_decay=1e-4)
    scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
                    optimizer, T_max=EPOCHS, eta_min=1e-6)

    best_val, best_state = 0., None
    print(f"\n{'Ep':>3} {'TrL':>7} {'TrA':>7} {'VL':>7} {'VA':>7}")
    print("─" * 38)

    for epoch in range(1, EPOCHS + 1):
        tr_l, tr_a = train_one_epoch(model, train_loader, optimizer, criterion)
        vl_l, vl_a = eval_loop(model, val_loader, criterion)
        scheduler.step()

        if vl_a > best_val:
            best_val   = vl_a
            best_state = {k: v.clone() for k, v in model.state_dict().items()}

        print(f"{epoch:>3d} {tr_l:>7.4f} {tr_a:>6.2f}% "
              f"{vl_l:>7.4f} {vl_a:>6.2f}%"
              f"{'  ★' if vl_a == best_val else ''}")

    model.load_state_dict(best_state)
    _, test_acc = eval_loop(model, test_loader, criterion)
    print(f"\nTest accuracy: {test_acc:.2f}%")
    n_params = sum(p.numel() for p in model.parameters())
    print(f"Parameters:    {n_params:,}")


if __name__ == '__main__':
    main()
```

### 17.3 Error Budget Analysis

Where does the remaining ~0.4–0.6% error come from in a well-trained CNN?

| Source | Fraction of errors |
|---|---|
| Genuinely ambiguous digits (even humans disagree) | ~30% |
| Unusual handwriting styles | ~25% |
| Specific class confusions (4↔9, 3↔8) | ~30% |
| Image quality issues (ink bleed, faint strokes) | ~15% |

Pushing from 99.3% to 99.8%+ requires: ensemble methods, test-time augmentation, very strong
data augmentation (elastic distortion), or larger architectures with pretraining.

## 18. Math Summary

### 18.1 All Key Equations

| Concept | Formula |
|---|---|
| **Normalisation** | $x_{\text{norm}} = (x - 0.1307)/0.3081$ |
| **Logistic regression** | $\hat{\mathbf{y}} = \text{softmax}(W\mathbf{x}+\mathbf{b})$ |
| **Softmax** | $\hat{p}_k = e^{z_k}/\sum_j e^{z_j}$ |
| **Cross-entropy** | $\mathcal{L} = -\log\hat{p}_c = -z_c + \log\sum_k e^{z_k}$ |
| **CE gradient** | $\partial\mathcal{L}/\partial z_k = \hat{p}_k - \mathbf{1}[k=c]$ |
| **Label smoothing** | $\tilde{y}_k = (1-\varepsilon)\mathbf{1}[k=c]+\varepsilon/K$ |
| **Focal loss** | $\mathcal{L}_f = -(1-\hat{p}_c)^\gamma\log\hat{p}_c$ |
| **MLP layer** | $\mathbf{h}^{(\ell)} = \text{ReLU}(\text{BN}(W^{(\ell)}\mathbf{h}^{(\ell-1)}+\mathbf{b}^{(\ell)}))$ |
| **BN (1D)** | $\hat{z}_j = (z_j-\mu_j)/\sqrt{\sigma_j^2+\epsilon}$, $y_j=\gamma_j\hat{z}_j+\beta_j$ |
| **Dropout** | $\tilde{h}_j = h_j/(1-p)$ w.p. $(1-p)$, else $0$ |
| **Conv output size** | $H_{\text{out}} = \lfloor(H+2P-k)/S\rfloor+1$ |
| **2D Conv** | $(X\star K)[i,j]=\sum_{p,q}K[p,q]\cdot X[i+p,j+q]$ |
| **Residual block** | $\mathbf{y} = \text{ReLU}(\mathcal{F}(\mathbf{x})+\mathbf{x})$ |
| **Kaiming init** | $\sigma_w^2 = 2/d_{\text{in}}$ |
| **L2 regularisation** | $\mathcal{L}_{\text{reg}} = \mathcal{L} + \frac{\lambda}{2}\|W\|^2$ |
| **Adam update** | $\theta \leftarrow \theta - \eta\hat{m}/(\sqrt{\hat{v}}+\epsilon)$ |
| **Cosine LR** | $\eta_t = \eta_{\min}+\frac{1}{2}(\eta_0-\eta_{\min})(1+\cos\pi t/T)$ |
| **Accuracy** | $\frac{1}{N}\sum_i\mathbf{1}[\hat{c}_i=c_i]$ |
| **Precision** | $C_{cc}/\sum_i C_{ic}$ |
| **Recall** | $C_{cc}/\sum_j C_{cj}$ |
| **F1** | $2PR/(P+R)$ |
| **ECE** | $\sum_m\frac{|B_m|}{N}|\text{acc}(B_m)-\text{conf}(B_m)|$ |
| **Saliency** | $|\partial\hat{p}_c/\partial x_i|$ |
